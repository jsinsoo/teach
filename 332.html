과학고 정보 수업: 목표지향적 개념기반 수업지도안

수업 주제: III. 알고리즘과 프로그래밍 - 332. 변수와 연산자: 데이터에 생명을 불어넣는 언어

수업 시작 멘트

위대한 물리학자 아르키메데스는 이렇게 말했습니다. "나에게 충분히 긴 지렛대와 받침점을 달라. 그러면 지구라도 들어 보이겠다." 프로그래밍에서 '변수'는 우리가 다룰 데이터라는 지구이며, '연산자'는 바로 그 지구를 들어 올리고, 쪼개고, 비교하고, 연결하는 강력한 지렛대와 받침점입니다. 오늘 우리는 이 강력한 도구들을 사용하여, 정적인 데이터에 생명을 불어넣고 우리가 원하는 결과로 가공하는 방법을 배우겠습니다.

1. 학습 목표

개념적 목표

  • 변수가 데이터(객체)를 가리키는 '이름'이며, 할당 연산자(=)가 연결의 의미임을 설명할 수 있다.
  • 산술, 비교, 논리 연산자가 각각 어떤 종류의 연산을 수행하고 어떤 타입의 결과를 반환하는지 명확히 구분할 수 있다.
  • 연산자 우선순위(Operator Precedence)의 개념을 이해하고, 괄호 ()를 사용하여 연산 순서를 명시적으로 제어하는 것의 중요성을 인지한다.

절차적 목표

  • 의미 있는 이름의 변수를 선언하고, 다양한 타입의 값을 할당 및 재할당할 수 있다.
  • 산술 연산자(+, -, *, /, //, %, **)를 사용하여 수학적 계산을 수행할 수 있다.
  • 비교 연산자(==, !=, >, < 등)와 논리 연산자(and, or, not)를 조합하여 복잡한 조건식을 만들 수 있다.
  • 복합 할당 연산자(+=, -= 등)를 사용하여 코드를 간결하게 작성할 수 있다.

태도적 목표

  • 변수 이름을 통해 코드의 의도를 명확하게 전달하려는 가독성 중심의 코딩 습관을 기른다.
  • 복잡한 수식을 코드로 옮길 때, 연산자 우선순위를 맹신하기보다 괄호를 사용하여 논리의 모호함을 제거하려는 신중한 태도를 갖는다.

2. 핵심 개념 상세 설명

변수 (Variable)와 할당 (Assignment)

  • 변수: 특정 데이터(객체)에 붙여진 '이름표'. 이 이름을 통해 프로그램은 메모리상의 데이터에 접근할 수 있습니다.
  • 할당 연산자 (=): "오른쪽의 값을 왼쪽의 변수 이름이 가리키게 하라"는 명령입니다. 수학의 '같다'가 아니라, '연결' 또는 '지정'의 의미입니다.

연산자 (Operator)

값에 대해 특정 연산(계산, 비교, 논리 조합 등)을 수행하도록 지시하는 기호입니다.

  • 산술 연산자 (Arithmetic Operators): 숫자 계산을 수행합니다.
    • +, -, *
    • /: 나눗셈 (결과는 항상 float)
    • //: 정수 나눗셈 (몫)
    • %: 나머지
    • **: 거듭제곱
  • 비교 연산자 (Comparison Operators): 두 값을 비교하여 True 또는 False를 결과로 냅니다.
    • == (같다), != (같지 않다), >, <, >=, <=
  • 논리 연산자 (Logical Operators): bool 값들을 조합하여 더 복잡한 논리를 만듭니다.
    • and (그리고), or (또는), not (부정)
  • 복합 할당 연산자 (Compound Assignment Operators): 연산과 할당을 한 번에 수행하여 코드를 간결하게 만듭니다. (예: x += 1은 x = x + 1과 동일)

연산자 우선순위 (Operator Precedence)

하나의 수식에 여러 연산자가 있을 때, 어떤 연산이 먼저 실행될지를 정해놓은 규칙입니다. 수학에서 곱셈/나눗셈이 덧셈/뺄셈보다 먼저 계산되는 것과 같습니다.

  • 일반적인 순서: ** (거듭제곱) → *, /, //, % → +, - → 비교 연산자 → 논리 연산자 (not → and → or)
  • 괄호 (): 우선순위 규칙을 무시하고, 괄호 안의 연산을 가장 먼저 수행하도록 강제합니다. 복잡한 수식에서는 우선순위를 외우기보다 괄호를 사용하여 의도를 명확히 하는 것이 훨씬 좋은 습관입니다.

3. 수업의 흐름 상세 기술

1단계: 도입 (5분) - 뉴턴의 만유인력 법칙

"뉴턴은 두 물체 사이의 만유인력이 F = G * (m1 * m2) / r^{2} 라는 위대한 법칙으로 표현될 수 있음을 발견했습니다. 이 공식 안에는 '곱셈', '나눗셈', '거듭제곱'과 같은 다양한 연산들이 정해진 순서에 따라 조화롭게 계산되어야만 올바른 힘의 크기를 구할 수 있습니다. 오늘 우리는 이처럼 데이터(질량, 거리)라는 재료를 연산자라는 도구로 요리하여, 새로운 정보(힘)를 만들어내는 과정을 배울 것입니다. 변수는 재료를 담는 그릇이며, 연산자는 요리 도구입니다."

2단계: 전개 (35분) - 데이터 요리하기

활동 1: 그릇과 도구 - 변수와 산술 연산자

변수 할당: mass1 = 5.972e24 (지구 질량), radius = 6371e3 (지구 반지름) 처럼, 변수가 어떻게 거대한 과학적 데이터를 담는 '그릇'이 되는지 보여줍니다.

산술 연산자: 지구 표면에서의 중력 가속도 g를 구하는 공식을 함께 코드로 작성합니다.


# 물리 상수 정의
G = 6.674e-11  # 만유인력 상수
M = 5.972e24   # 지구 질량 (kg)
R = 6371e3     # 지구 반지름 (m)

# 중력 가속도 계산: g = G * M / R^2
g_surface = G * M / (R ** 2)

print(f"지구 표면 중력 가속도 계산 결과: {g_surface:.2f} m/s^2")
"이 코드에서 **가 /나 *보다 먼저 계산되고, 괄호 ()가 연산 순서를 어떻게 명확하게 하는지 주목하세요. 괄호가 없다면 의도와 다른 계산이 될 수 있습니다."

활동 2: 진실 혹은 거짓 - 비교와 논리 연산자

비교 연산: "어떤 물체의 속도가 지구의 탈출 속도(약 11.2 km/s)보다 큰가?"


object_speed = 15.0 # km/s
ESCAPE_VELOCITY = 11.2

can_escape = object_speed >= ESCAPE_VELOCITY
print(f"이 물체는 탈출 가능한가? {can_escape}")
print(f"can_escape 변수의 타입: {type(can_escape)}") # bool 타입임을 확인

논리 연산: "이 행성이 '생명체 거주 가능 영역(Habitable Zone)'에 있으려면, 행성의 온도가 0도에서 100도 '사이여야 하고', 물이 '존재해야 한다'."


planet_temp = 15.0
water_exists = True

# and 연산자를 사용하여 두 조건을 모두 만족하는지 확인
is_habitable = (0 < planet_temp < 100) and water_exists
print(f"이 행성은 거주 가능한가? {is_habitable}")
"파이썬의 0 < planet_temp < 100 와 같은 연쇄 비교와, and 논리 연산자를 조합하여 복잡한 과학적 조건을 어떻게 간결한 코드로 표현하는지 확인해봅시다."

활동 3: 간결함의 미학 - 복합 할당 연산자

for 루프를 이용한 누적 합계 예제를 다시 보여줍니다.


total = 0
for i in range(1, 101):
    # total = total + i 와 동일
    total += i
print("1부터 100까지의 합:", total)
"total = total + i 보다 total += i 가 더 짧고, 'total에 i를 더해 누적시킨다'는 의도를 더 명확하게 보여줍니다. 코드를 간결하게 만드는 좋은 습관입니다."

3단계: 정리 (5분) - 표현식의 구성

"모든 복잡한 계산과 조건은 결국, '변수(데이터)'와 '연산자(기능)'가 결합된 '표현식(Expression)'으로 이루어집니다."

산술, 비교, 논리 연산자가 각각 숫자, bool 값을 어떻게 만들어내고, 이들이 다시 if문이나 다른 변수 할당에 어떻게 사용되는지 그 흐름을 요약합니다. "변수와 연산자는 여러분의 논리를 표현하는 가장 기본적인 언어입니다."

수업 마무리 멘트

수학자이자 철학자였던 버트런드 러셀은 말했습니다. "수학은 올바르게 바라보면, 지고의 아름다움을 지니고 있다. 조각품처럼 차갑고 엄격한 아름다움을." 여러분이 오늘 배운 연산자와 변수는 바로 이 수학적 논리의 아름다움을 코드의 세계에 구현하는 도구입니다. 이 도구들을 사용하여, 여러분의 생각을 차갑고 엄격한 논리로, 동시에 명료하고 아름다운 코드로 빚어내시길 바랍니다.

4. 진단 및 형성평가

진단평가 (OX 퀴즈 10개)

1. x = 10 코드에서 =는 'x와 10이 같다'는 수학적 의미이다. (X)

풀이: 프로그래밍에서 =는 '할당(assignment)' 연산자입니다. 오른쪽의 값(10)을 왼쪽의 변수(x)가 가리키도록 연결하는 '명령'입니다. 수학적 동등 비교는 == 연산자를 사용합니다.

2. 10 / 3의 결과와 10 // 3의 결과는 같다. (X)

풀이: /는 일반 나눗셈으로, 결과는 실수 3.333... (float) 입니다. //는 정수 나눗셈(몫)으로, 소수점 이하를 버린 정수 3 (int) 입니다. 두 연산자는 목적과 결과 타입이 다릅니다.

3. x = 10일 때, x == 10의 결과는 True이다. (O)

풀이: ==는 비교 연산자로, 양쪽의 값이 같은지 확인합니다. 변수 x는 10을 가리키고 있으므로, 10과 비교하면 True를 반환합니다.

4. True and False의 결과는 True이다. (X)

풀이: and 연산자는 양쪽 모두가 True일 때만 True를 반환합니다. 하나라도 False가 있으면 결과는 False가 됩니다.

5. 2 ** 3은 2의 3제곱, 즉 8을 의미한다. (O)

풀이: **는 거듭제곱 연산자입니다. 2 ** 3은 2를 3번 곱한 2 * 2 * 2와 같으며, 결과는 8입니다.

6. count = count + 1은 count += 1과 완전히 동일한 의미이다. (O)

풀이: +=는 복합 할당 연산자로, 변수 = 변수 + 값의 축약형입니다. 기능적으로 완전히 동일하며, 코드를 더 간결하게 만들어줍니다.

7. 2 + 3 * 4의 결과는 20이다. (X)

풀이: 연산자 우선순위에 따라 곱셈(*)이 덧셈(+)보다 먼저 실행됩니다. 따라서 3 * 4가 먼저 계산되어 12가 되고, 그 다음 2 + 12가 계산되어 최종 결과는 14가 됩니다. 20이 되려면 (2 + 3) * 4와 같이 괄호를 사용해야 합니다.

8. not True의 결과는 False이다. (O)

풀이: not 연산자는 불리언 값을 반대로 뒤집는 역할을 합니다. 따라서 not True는 False가 되고, not False는 True가 됩니다.

9. 변수 이름은 숫자로 시작할 수 있다. (예: 1st_variable = 1) (X)

풀이: 파이썬의 변수 명명 규칙에 따라, 변수 이름은 영문자나 언더스코어(_)로 시작해야 합니다. 숫자는 이름의 두 번째 글자부터 사용할 수 있습니다.

10. x = 10일 때, 5 < x < 15는 올바른 파이썬 문법이다. (O)

풀이: 파이썬은 다른 언어와 달리 수학에서처럼 비교 연산자를 연결하여 범위를 표현하는 것을 허용합니다. 이 코드는 내부적으로 5 < x and x < 15와 동일하게 동작합니다.

형성평가 (5지선다형 10개)

1. 다음 코드의 실행 결과로 올바른 것은? print(17 // 5, 17 % 5)

  1. 3.4 2
  2. 3 2
  3. 3.0 2.0
  4. 4 2
  5. 3 2.0
정답 및 해설: ② (17 // 5는 17을 5로 나눈 몫의 정수 부분인 3입니다. 17 % 5는 17을 5로 나눈 나머지인 2입니다. print 함수는 이 두 값을 공백으로 구분하여 출력하므로 3 2가 됩니다.)

2. year = 2024일 때, 'year가 4의 배수이고 100의 배수는 아니다'를 올바르게 표현한 코드는?

  1. year % 4 == 0 or year % 100 != 0
  2. year // 4 == 0 and year // 100 != 0
  3. year % 4 == 0 and year % 100 != 0
  4. year % 4 == 0 and not year % 100 == 0
  5. year / 4 == 0 && year / 100 != 0
정답 및 해설: ③ ('A이고 B이다'는 A and B로 표현합니다. '4의 배수'는 4로 나눈 나머지가 0이라는 의미이므로 year % 4 == 0입니다. '100의 배수가 아니다'는 100으로 나눈 나머지가 0이 아니라는 의미이므로 year % 100 != 0입니다. 이 둘을 and로 연결한 ③이 가장 직접적인 표현입니다. ④도 논리적으로는 같습니다.)

3. x = 5일 때, 다음 중 결과가 False인 것은?

  1. x > 3 and x < 10
  2. x == 5 or x == 10
  3. not x == 10
  4. x % 2 == 0
  5. x ** 2 == 25
정답 및 해설: ④ (① True and True -> True. ② True or False -> True. ③ not False -> True. ④ 5 % 2는 1이므로 1 == 0은 False. ⑤ 5 ** 2는 25이므로 25 == 25는 True입니다.)

4. 다음 코드의 실행 결과는 무엇인가? print(2 + 3 * 4 ** 2)

  1. 14
  2. 20
  3. 50
  4. 100
  5. 400
정답 및 해설: ③ (연산자 우선순위에 따라 거듭제곱(**)이 가장 먼저, 그 다음 곱셈(*), 마지막으로 덧셈(+)이 실행됩니다. 1. 4 ** 2 = 16, 2. 3 * 16 = 48, 3. 2 + 48 = 50)

5. 다음 중 =와 ==의 사용법이 모두 올바르게 사용된 코드는?

  1. if (x = 10): print("x는 10")
  2. x == 10
  3. if x == 10: x = x + 1
  4. if x = y: print("같음")
  5. print(x = 10)
정답 및 해설: ③ (if문의 조건 부분에는 비교 연산자 ==를 사용하여 x가 10과 같은지 물어야 합니다. if 블록 안에서는 할당 연산자 =를 사용하여 x의 값을 변경할 수 있습니다. ③은 이 두 가지를 올바르게 사용하고 있습니다.)

6. "온도가 0도 미만이거나 100도 이상이면 '비정상'" 이라는 조건을 나타내는 코드로 올바른 것은? (temp = 120)

  1. if temp < 0 and temp >= 100:
  2. if temp < 0 or temp >= 100:
  3. if not (temp >= 0 or temp < 100):
  4. if 0 < temp < 100:
  5. if temp != 0 and temp != 100:
정답 및 해설: ② ('~이거나'는 논리합을 의미하는 or 연산자로 표현해야 합니다. 따라서 '온도가 0 미만'(temp < 0) or '온도가 100 이상'(temp >= 100)을 연결한 ②가 정확한 표현입니다.)

7. 변수 score에 85가 저장되어 있다. score /= 10 연산을 수행한 후 score의 값은?

  1. 8
  2. 8.5
  3. 9
  4. 850
  5. 75

a = int(input("첫 번째 정수: "))

문제 4: (복합 할당 연산자) for 루프와 복합 할당 연산자를 사용하여, 1부터 100까지의 숫자 중 3의 배수들의 합계를 구하세요.

total = 0
for i in range(1, 101):
    if i % 3 == 0:
        total += i  # total = total + i 와 동일

print("1부터 100까지 3의 배수의 합:", total)
상세한 풀이와 설명: 누적 합계를 구할 때, total = total + i보다 더 간결한 total += i를 사용하는 연습입니다. 코드의 길이를 줄이고, '누적한다'는 의도를 더 명확히 보여줍니다.

문제 5: (연산자 우선순위) 섭씨온도 C를 화씨온도 F로 변환하는 공식은 F = C * 9/5 + 32 입니다. 섭씨 30도를 화씨로 변환하여 출력하세요. 괄호가 없어도 올바른 결과가 나오는 이유를 주석으로 설명하세요.

celsius = 30

# 곱셈/나눗셈이 덧셈보다 우선순위가 높으므로,
# (celsius * 9 / 5) + 32 와 같이 계산됨.
fahrenheit = celsius * 9 / 5 + 32

print(f"섭씨 {celsius}도는 화씨 {fahrenheit:.1f}도 입니다.")
상세한 풀이와 설명: 파이썬의 연산자 우선순위 규칙에 따라, *와 /가 +보다 먼저 실행됩니다. 따라서 celsius * 9 / 5가 먼저 계산되고, 그 결과에 32가 더해지므로 괄호가 없어도 올바른 결과를 냅니다. 하지만 복잡한 수식에서는 가독성을 위해 괄호를 사용하는 것이 좋습니다.

문제 6: (변수 교환) 두 변수 a와 b의 값을 서로 바꾸는 코드를, 임시 변수를 사용하는 방법과 파이썬의 동시 할당을 사용하는 방법 두 가지로 모두 작성하세요.

a, b = 10, 20
print(f"바꾸기 전: a={a}, b={b}")

# 방법 1: 임시 변수 사용
temp = a
a = b
b = temp
print(f"방법 1 후: a={a}, b={b}")

# 다시 초기화 후 방법 2 실행
a, b = 10, 20
# 방법 2: 파이썬 동시 할당 (튜플 언패킹)
a, b = b, a
print(f"방법 2 후: a={a}, b={b}")
상세한 풀이와 설명: 값 교환(swap)은 매우 흔한 알고리즘 패턴입니다. 전통적인 방법은 temp라는 임시 그릇을 사용하여 한쪽 값을 잠시 보관하는 것입니다. 파이썬에서는 a, b = b, a 와 같이 한 줄로 더 간결하게 표현할 수 있는데, 이는 내부적으로 (b, a)라는 튜플을 만들어 각 변수에 다시 할당하는 방식으로 동작합니다.

문제 10: (f-string과 연산) 반지름이 5인 구의 겉넓이(4 * \pi * r^{2})와 부피(4/3 * \pi * r^{3})를 계산하고, f-string을 사용하여 소수점 셋째 자리까지 결과를 출력하세요. (math.pi 사용)

import math

radius = 5

# f-string의 중괄호 안에서 직접 계산 수행

상세한 풀이와 설명: f-string의 강력한 기능입니다. 중괄호 {} 안에는 변수뿐만 아니라, 복잡한 산술 표현식도 넣을 수 있습니다. 파이썬은 이 표현식을 먼저 계산한 뒤, 그 결과를 문자열에 삽입합니다. :.3f는 결과를 소수점 셋째 자리까지 형식화하여 출력하라는 의미입니다.
