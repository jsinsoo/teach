과학고 정보 수업 지도안: 문제와 문제 해결 절차

III. 알고리즘과 프로그래밍 - 312. 생각의 설계도를 그리는 법

1. 학습 목표

개념적 목표

  • 알고리즘이 '문제 해결을 위한 명확하고 유한한 절차의 집합'임을 정의할 수 있다.
  • '문제 이해 → 해결 전략 수립 → 실행 → 반성'으로 이어지는 문제 해결의 4단계를 설명할 수 있다.
  • 복잡한 문제를 더 작고 해결 가능한 단위로 나누는 '문제 분해(Decomposition)'의 중요성을 이해한다.
  • 의사코드(Pseudocode)가 특정 프로그래밍 언어에 얽매이지 않고 알고리즘의 논리를 표현하는 효과적인 도구임을 설명할 수 있다.

절차적 목표

  • 주어진 문제에 대해 해결 절차 4단계(문제 이해, 전략 수립, 실행, 반성)를 적용할 수 있다.
  • 문제의 입력(Input), 출력(Output), 제약 조건(Constraints)을 명확히 정의할 수 있다.
  • 문제 해결 전략을 의사코드나 순서도로 표현하고, 이를 파이썬 코드로 구현할 수 있다.

태도적 목표

  • 문제를 마주했을 때, 즉시 코딩을 시작하기보다 문제의 본질을 분석하고 해결 절차를 먼저 설계하려는 계획적인 자세를 갖는다.
  • 논리적으로 명확하고 효율적인 해결책을 찾으려는 공학적 사고방식을 기른다.
  • 자신의 알고리즘이 틀릴 수 있음을 인정하고, 다양한 테스트 케이스를 통해 검증하려는 과학적이고 비판적인 태도를 갖는다.

2. 핵심 개념 상세 설명

문제 해결 4단계 (조지 폴리아의 4단계)

수학자 조지 폴리아(George Pólya)가 제시한 문제 해결 방법론으로, 프로그래밍 문제 해결에도 매우 효과적으로 적용됩니다.

  1. 1단계: 문제 이해 (Understanding the Problem): 무엇을 구해야 하는가? 주어진 데이터(입력)는 무엇인가? 조건(제약사항)은 무엇인가? 용어의 의미를 명확히 이해하고, 문제를 자신의 언어로 다시 설명해보는 단계입니다.
  2. 2단계: 해결 전략 수립 (Devising a Plan): 입력과 출력 사이의 관계를 어떻게 찾을 것인가? 문제를 더 작은 단위로 나눌 수 있는가(문제 분해)? 비슷한 문제를 풀어본 경험이 있는가? 해결을 위한 구체적인 절차, 즉 알고리즘을 설계하는 단계입니다. 의사코드나 순서도가 이때 사용됩니다.
  3. 3단계: 실행 (Carrying out the Plan): 수립된 전략(알고리즘)을 실제 코드로 구현하는 단계입니다. 각 단계가 올바른지 검증하며 신중하게 코드를 작성합니다.
  4. 4단계: 반성 (Looking Back): 결과가 올바른가? 더 효율적인 다른 방법은 없는가? 이 해결 방법을 다른 문제에도 적용할 수 있는가? 코드를 테스트하고, 리팩토링하며, 해결 과정을 통해 배운 점을 정리하는 단계입니다.

추상화 (Abstraction)와 모델링 (Modeling)

  • 추상화: 문제의 복잡한 현실 세계에서 불필요한 세부 사항은 제거하고, 문제 해결에 필요한 '핵심적인 특징'에만 집중하는 과정입니다. (예: 지도를 그릴 때, 모든 건물을 다 그리지 않고 중요한 도로와 랜드마크만 표시하는 것)
  • 모델링: 추상화를 통해 파악한 핵심 특징들을, 컴퓨터가 이해할 수 있는 형태(변수, 자료구조, 클래스 등)로 표현하고 구조화하는 과정입니다.

의사코드 (Pseudocode)

'pseudo'는 '가짜의, 유사한'이라는 뜻으로, 실제 프로그래밍 언어의 엄격한 문법에 얽매이지 않고, 자연어와 프로그래밍 언어의 문법을 혼합하여 알고리즘의 논리를 표현하는 방식입니다. 코드를 작성하기 전에 생각을 정리하고, 언어에 상관없이 다른 사람과 알고리즘에 대해 소통하는 데 매우 유용합니다.

3. 수업의 흐름

(도입) 길 잃은 탐험가 (5분)

"만약 내가 문제를 해결하는 데 한 시간이 주어진다면, 나는 55분을 문제를 올바르게 정의하는 데 쓰고, 나머지 5분을 해결책을 찾는 데 쓸 것이다." - 알베르트 아인슈타인

천재 물리학자 아인슈타인의 말처럼, 코드를 한 줄이라도 작성하기 전에, 문제를 철저히 이해하고, 해결 전략을 세우며, 명확한 계획을 그리는 '문제 해결 절차'가 무엇보다 중요합니다. 훌륭한 건축이 훌륭한 설계도에서 시작되듯, 훌륭한 프로그램은 훌륭한 생각의 절차에서 시작됩니다. 오늘 우리는 성공적인 탐험을 위한 '지도와 나침반', 즉 체계적인 문제 해결 절차를 배우겠습니다.

(전개) '가짜 동전 찾기' 프로젝트 (35분)

활동 1: 문제의 명확한 이해

문제 제시: "똑같이 생긴 8개의 동전이 있다. 이 중 단 하나의 동전만 다른 동전들보다 가볍다. 양팔 저울을 '최소한'으로 사용하여 가짜 동전을 찾아내라."

학생들과의 문답:

  • 우리가 찾아야 할 최종 결과(Output)는 무엇인가? → 가짜 동전의 번호(인덱스)
  • 주어진 재료(Input)는 무엇인가? → 8개의 동전
  • 사용할 수 있는 도구와 규칙(Constraints)은 무엇인가? → 양팔 저울, 무게 비교만 가능(가볍다, 같다, 무겁다), 저울 사용 횟수를 최소화해야 함.
이처럼 입력, 출력, 제약 조건을 명확히 정의하는 것이 모든 문제 해결의 출발점입니다.

활동 2: 해결 전략 수립 (알고리즘 설계)

학생들의 아이디어를 '분할 정복(Divide and Conquer)' 전략으로 유도합니다. "어떻게 하면 저울 한 번으로 가장 많은 정보를 얻을 수 있을까요?"

전략: "8개의 동전을 (3개, 3개, 2개) 세 그룹으로 나눕니다. 그리고 3개짜리 두 그룹을 저울에 올립니다."

의사코드 작성:


1. 동전 8개를 A(1,2,3), B(4,5,6), C(7,8) 그룹으로 나눈다.
2. 저울에 A와 B를 올린다.
3. 만약 저울이 수평이면 (A와 B의 무게가 같으면):
   가짜 동전은 그룹 C에 있다. -> C그룹의 7번, 8번을 저울에 올려 가벼운 쪽을 찾는다. (저울 2번 사용)
4. 만약 A쪽이 올라가면 (A가 더 가벼우면):
   가짜 동전은 그룹 A에 있다. -> A그룹의 1번, 2번을 저울에 올려 비교한다.
   만약 수평이면, 남은 3번이 가짜다.
   만약 한쪽이 가벼우면, 그쪽이 가짜다. (저울 2번 사용)
5. 만약 B쪽이 올라가면 (B가 더 가벼우면):
   가짜 동전은 그룹 B에 있다. -> B그룹의 4번, 5번을 저울에 올려 위와 같이 비교한다. (저울 2번 사용)

활동 3: 계획 실행 (Python 코드 구현)

위 의사코드를 파이썬 코드로 함께 번역합니다.


def find_fake_coin(coins):
    # coins는 8개짜리 리스트, 가짜는 0.9, 진짜는 1.0 이라고 가정
    # 1. A(0,1,2), B(3,4,5)를 저울에 올림
    if sum(coins[0:3]) == sum(coins[3:6]):
        print("1차 계량: 그룹 A와 B의 무게가 같습니다.")
        # 2. 그룹 C(6,7)를 저울에 올림
        if coins[6] < coins[7]: return 6
        else: return 7
    elif sum(coins[0:3]) < sum(coins[3:6]):
        print("1차 계량: 그룹 A가 더 가볍습니다.")
        # 3. 그룹 A(0,1,2) 내에서 비교
        if coins[0] == coins[1]: return 2
        elif coins[0] < coins[1]: return 0
        else: return 1
    else: # 그룹 B가 더 가벼울 때
        print("1차 계량: 그룹 B가 더 가볍습니다.")
        # 그룹 B 내에서 위와 동일한 로직
        if coins[3] == coins[4]: return 5
        elif coins[3] < coins[4]: return 3
        else: return 4
의사코드의 논리 구조가 어떻게 파이썬의 if-elif-else 제어 구조로 그대로 옮겨지는지 확인하세요. 좋은 설계는 좋은 코드로 이어집니다.

활동 4: 반성 및 회고

  • 검증: "우리가 만든 알고리즘은 저울을 최대 몇 번 사용했나요?" → 2번. "최소 횟수가 맞을까요?" (3^{1} < 8 \leq 3^{2}, 따라서 2번이면 충분함을 설명)
  • 확장: "만약 동전이 9개라면? 27개라면? 이 알고리즘을 일반화할 수 있을까요?"
  • 다른 해결책: "만약 가짜 동전이 더 무거울 수도, 가벼울 수도 있다면 알고리즘은 어떻게 바뀌어야 할까요?"
하나의 해결책에 만족하지 않고, 결과를 검증하고, 더 나은 방법을 고민하며, 다른 문제에 적용해보려는 '반성'의 단계가 여러분을 더 뛰어난 문제 해결사로 만들어 줄 것입니다.

(마무리) 생각의 습관 (5분)

"나는 생각한다, 고로 존재한다." - 르네 데카르트

훌륭한 프로그래머는 '생각 없는 코더'가 아니라 '생각하는 철학자'와 같습니다. 오늘 배운 문제 해결 4단계라는 강력한 사유의 도구를 사용하여, 여러분 앞에 놓인 어떤 복잡한 문제라도 명쾌하게 풀어내는 디지털 시대의 데카르트가 되시길 바랍니다.

4. 진단 및 형성평가

OX 퀴즈

1. 알고리즘은 문제를 해결하기 위한 단계적인 절차를 의미하며, 반드시 컴퓨터 프로그램으로 작성되어야 한다. (X)

풀이: 알고리즘은 생각의 절차이며, 컴퓨터 프로그램뿐만 아니라 요리 레시피, 길 찾기 안내처럼 일상생활의 여러 형태로 존재할 수 있습니다.

2. 의사코드(Pseudocode)는 특정 프로그래밍 언어의 문법에 맞춰 엄격하게 작성해야 한다. (X)

풀이: 의사코드는 특정 언어에 얽매이지 않고, 자연어와 유사한 형태로 알고리즘의 논리를 자유롭게 표현하는 것이 목적입니다.

3. 좋은 알고리즘은 어떤 경우에라도 유한한 시간 안에 반드시 종료되어야 한다. (O)

풀이: 이는 알고리즘의 '유한성(Finiteness)' 조건입니다. 무한히 반복하거나 끝나지 않는 절차는 올바른 알고리즘이라고 할 수 없습니다.

4. '문제 분해'는 하나의 큰 문제를 여러 개의 작은 함수로 나누어 코딩하는 기술을 의미한다. (X)

풀이: 문제 분해는 코딩 이전의 '사고 과정'입니다. 복잡한 문제를 논리적으로 더 작고 관리하기 쉬운 단위로 나누어 생각하는 것을 의미합니다.

5. 알고리즘의 효율성은 주로 '최선의 경우(Best Case)'를 기준으로 평가한다. (X)

풀이: 알고리즘의 성능을 보장하기 위해, 일반적으로 '최악의 경우(Worst Case)'를 기준으로 효율성을 평가합니다. (Big-O 표기법)

6. 유클리드 호제법은 두 수의 최소공배수(LCM)를 찾는 알고리즘이다. (X)

풀이: 유클리드 호제법은 두 수의 '최대공약수(GCD)'를 찾는 매우 효율적인 고전 알고리즘입니다.

7. 알고리즘의 각 단계는 누가 보더라도 모호하지 않고 명확하게 이해될 수 있어야 한다. (O)

풀이: 이는 알고리즘의 '명확성(Definiteness)' 조건입니다.

8. '선형 탐색'은 정렬되지 않은 리스트에서도 원하는 값을 찾을 수 있다. (O)

풀이: 선형 탐색은 리스트의 처음부터 끝까지 모든 요소를 하나씩 순서대로 확인하는 방식이므로, 데이터가 정렬되어 있지 않아도 사용할 수 있습니다.

9. 알고리즘과 프로그램은 완전히 동일한 개념이다. (X)

풀이: 알고리즘은 '방법' 또는 '아이디어'이고, 프로그램은 그 알고리즘을 특정 프로그래밍 언어를 사용하여 '구현'한 실체입니다.

10. 알고리즘을 설계하는 가장 중요한 이유는 코드의 줄 수를 줄이기 위해서이다. (X)

풀이: 가장 중요한 이유는 문제를 '정확하고' '효율적으로' 해결하기 위해서입니다. 좋은 알고리즘이 간결한 코드로 이어지는 경우가 많지만, 코드의 길이 자체가 목적은 아닙니다.

5지선다형 퀴즈

1. 다음 중 '알고리즘'에 대한 정의로 가장 적절한 것은?

  1. 파이썬으로 작성된 코드의 묶음이다.
  2. 문제를 해결하기 위한 명확하고 단계적인 절차나 규칙의 집합이다.
  3. 컴퓨터 하드웨어를 제어하는 명령어이다.
  4. 데이터의 종류를 나타내는 자료형이다.
  5. 프로그램의 실행 속도를 측정하는 단위이다.
정답 및 해설: ② (알고리즘은 특정 언어나 기술에 국한되지 않는, 문제 해결을 위한 논리적인 '방법론' 그 자체를 의미합니다.)

2. 알고리즘을 코드로 구현하기 전에 의사코드(Pseudocode)를 작성하는 주된 이점은?

  1. 프로그램의 실행 속도가 빨라진다.
  2. 특정 프로그래밍 언어의 문법에 얽매이지 않고 문제 해결의 논리적 흐름에만 집중할 수 있다.
  3. 코드가 자동으로 생성된다.
  4. 모든 문법적 오류를 미리 발견할 수 있다.
  5. 메모리 사용량을 예측할 수 있다.

문제 1: (합계 알고리즘) 주어진 숫자 리스트의 모든 요소의 합계를 구하는 알고리즘을 함수 `calculate_sum(numbers)`로 구현하세요. (sum() 내장 함수 사용 금지)


def calculate_sum(numbers):
    # 1. 합계를 저장할 누적 변수를 0으로 초기화
    total = 0
    # 2. 리스트의 모든 숫자를 하나씩 순회
    for num in numbers:
        # 3. 현재 숫자를 누적 변수에 더함
        total += num
    # 4. 최종 합계를 반환
    return total

my_list = [10, 20, 30, 40]
print(f"리스트의 합계: {calculate_sum(my_list)}") # 출력: 100

문제 2: (평균 알고리즘) 주어진 숫자 리스트의 평균을 구하는 `calculate_average(numbers)` 함수를 구현하세요. (위 `calculate_sum` 함수와 `len()` 함수 활용)


def calculate_average(numbers):
    # 예외 처리: 리스트가 비어있으면 0으로 나눌 수 없음
    if not numbers:
        return 0
    # 1. 합계를 구한다 (문제 분해)
    total = calculate_sum(numbers)
    # 2. 개수를 구한다
    count = len(numbers)
    # 3. 합계를 개수로 나눈다
    return total / count

print(f"리스트의 평균: {calculate_average(my_list)}") # 출력: 25.0

문제 3: (최댓값 알고리즘) 주어진 숫자 리스트에서 가장 큰 값을 찾아 반환하는 `find_max(numbers)` 함수를 구현하세요. (max() 내장 함수 사용 금지)


def find_max(numbers):
    if not numbers:
        return None # 빈 리스트는 최댓값이 없음
    # 1. 첫 번째 값을 현재까지의 최댓값으로 가정
    max_value = numbers[0]
    # 2. 두 번째 값부터 끝까지 순회
    for num in numbers[1:]:
        # 3. 더 큰 값을 만나면 최댓값 변수를 갱신
        if num > max_value:
            max_value = num
    # 4. 최종 최댓값을 반환
    return max_value

my_list = [5, 2, 9, 1, 7]
print(f"가장 큰 값: {find_max(my_list)}") # 출력: 9

문제 4: (팩토리얼 알고리즘) 양의 정수 n을 입력받아 n의 팩토리얼(n!)을 계산하여 반환하는 `factorial(n)` 함수를 구현하세요. (팩토리얼: 1부터 n까지의 모든 정수의 곱)


def factorial(n):
    if n < 0: return "음수는 정의되지 않음"
    if n == 0: return 1
    # 1. 곱셈 누적 변수는 1로 초기화
    product = 1
    # 2. 1부터 n까지 순회
    for i in range(1, n + 1):
        # 3. 현재 숫자를 누적 변수에 곱함
        product *= i
    # 4. 최종 곱을 반환
    return product

print(f"5의 팩토리얼: {factorial(5)}") # 출력: 120
상세한 풀이와 설명: 곱셈 누적 알고리즘입니다. 합계와 달리, 곱셈의 항등원은 1이므로 누적 변수를 반드시 1로 초기화해야 합니다.

문제 5: (선형 탐색 알고리즘) 숫자 리스트 data와 찾으려는 값 target을 인자로 받아, target이 리스트에 존재하면 그 인덱스를, 존재하지 않으면 -1을 반환하는 `linear_search(data, target)` 함수를 구현하세요.


def linear_search(data, target):
    # enumerate를 사용하여 인덱스와 값을 함께 가져옴
    for index, value in enumerate(data):
        if value == target:
            return index # 값을 찾으면 즉시 인덱스를 반환하고 종료
    return -1 # 루프를 다 돌았는데도 못 찾았으면 -1 반환

my_list = [1, 5, 2, 9, 7]
print(f"9의 위치: {linear_search(my_list, 9)}") # 출력: 3
print(f"3의 위치: {linear_search(my_list, 3)}") # 출력: -1
상세한 풀이와 설명: `enumerate` 함수를 사용하면 리스트를 순회할 때 인덱스와 값을 편리하게 함께 얻을 수 있습니다. 값을 찾는 즉시 `return`을 통해 함수를 종료하여 불필요한 반복을 막는 것이 효율적입니다.

문제 8: (소수 판별 알고리즘) 자연수 n을 입력받아, 그 수가 소수이면 True를, 아니면 False를 반환하는 `is_prime(n)` 함수를 구현하세요.


def is_prime(n):
    if n < 2: # 2보다 작은 수는 소수가 아님
        return False
    # 2부터 n-1까지 확인하는 기본 알고리즘
    # (최적화: 2부터 n의 제곱근까지만 확인해도 됨)
    for i in range(2, n):
        if n % i == 0:
            return False # 나누어 떨어지는 수가 있으면 소수가 아님
    # 루프를 무사히 통과하면 소수임
    return True

print(f"17은 소수인가? {is_prime(17)}") # 출력: True
print(f"18은 소수인가? {is_prime(18)}") # 출력: False
상세한 풀이와 설명: 소수의 정의('1과 자기 자신 외에는 나누어 떨어지지 않는 수')를 그대로 알고리즘으로 옮긴 것입니다. for 루프로 2부터 n-1까지 나누어 보다가, 나누어 떨어지는 약수가 하나라도 발견되면 즉시 False를 반환하고 함수를 종료합니다.

문제 9: (피보나치 수열 알고리즘) 양의 정수 n을 입력받아, 피보나치 수열의 n번째 항을 구하는 `fibonacci(n)` 함수를 구현하세요. (시작: 1, 1, 2, 3, 5, ...)


def fibonacci(n):
    if n <= 2:
        return 1
    # a는 이전 항, b는 현재 항을 나타내는 상태 변수
    a, b = 1, 1
    # 3번째 항부터 n번째 항까지 반복
    for _ in range(n - 2):
        # 다음 항 계산: a는 b가 되고, b는 a+b가 됨
        a, b = b, a + b
    return b

print(f"피보나치 수열의 10번째 항: {fibonacci(10)}") # 출력: 55
상세한 풀이와 설명: 두 개의 상태 변수(a, b)를 사용하여 수열의 현재 상태를 기억하고, 반복을 통해 상태를 계속 업데이트하는 방식의 알고리즘입니다. 파이썬의 동시 할당 `a, b = b, a + b`를 사용하면 임시 변수 없이도 간결하게 구현할 수 있습니다.

문제 10: (선택 정렬 알고리즘) 숫자 리스트를 오름차순으로 정렬하는 `selection_sort(data)` 함수를 구현하세요. (내장 sorted() 함수 사용 금지)


def selection_sort(data):
    # 원본을 바꾸지 않기 위해 리스트를 복사
    sorted_data = data[:]
    n = len(sorted_data)
    for i in range(n - 1):
        # 현재 위치 i를 최솟값의 위치로 가정
        min_index = i
        # i 다음 위치부터 끝까지 순회하며 진짜 최솟값 위치를 찾음
        for j in range(i + 1, n):
            if sorted_data[j] < sorted_data[min_index]:
                min_index = j
        # 찾은 최솟값을 현재 위치 i와 교환
        sorted_data[i], sorted_data[min_index] = sorted_data[min_index], sorted_data[i]
    return sorted_data

my_list = [5, 2, 9, 1, 7]
print(f"정렬 전: {my_list}")
print(f"정렬 후: {selection_sort(my_list)}") # 출력: [1, 2, 5, 7, 9]
상세한 풀이와 설명: 선택 정렬은 대표적인 정렬 알고리즘 중 하나입니다. 1. 아직 정렬되지 않은 부분에서 가장 작은 요소를 찾는다. 2. 그 요소를 정렬되지 않은 부분의 맨 첫 번째 요소와 교환한다. 3. 정렬된 부분을 하나 늘리고, 위 과정을 반복한다. 이 논리를 중첩 for 루프와 변수 교환을 통해 구현한 것입니다. 알고리즘적 사고를 연습하는 좋은 예제입니다.
