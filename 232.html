과학고 정보 수업 지도안: II. 데이터 - 232. 문제 해결에 필요한 데이터

1. 학습 목표

개념적 목표:

  • 데이터 기반 문제 해결의 첫 단계가 '풀고자 하는 문제를 명확히 정의'하는 것임을 설명할 수 있다.
  • 정의된 문제를 해결하기 위해 어떤 종류의 데이터(변수, 속성)가 필요한지 '데이터 요구사항'을 정의할 수 있다.
  • 데이터 수집의 다양한 방법(센서, 로그, API, 웹 스크래핑 등)을 이해하고, 각 방법의 특징을 설명할 수 있다.
  • 데이터의 품질을 결정하는 주요 요소(정확성, 완전성, 일관성)를 이해하고, 데이터 정제(Data Cleaning)의 필요성을 설명할 수 있다.
  • 탐색적 데이터 분석(EDA)의 목적이 데이터를 본격적으로 분석하기 전에 데이터의 기본적인 특성과 구조를 파악하는 것임을 이해한다.

절차적 목표:

  • 주어진 문제 시나리오에 대해, 필요한 데이터 항목과 그 데이터 타입을 구체적으로 나열할 수 있다.
  • Python 코드를 사용하여 웹 페이지의 특정 정보를 가져오는 간단한 웹 스크래핑(Web Scraping) 과정을 시뮬레이션할 수 있다.
  • 데이터셋에 포함된 결측치(Missing Value)나 이상치(Outlier)를 식별하고, 이를 제거하거나 특정 값으로 대체하는 데이터 정제 코드를 작성할 수 있다.
  • 수집된 데이터에 대해 최소/최대/평균값 등 기초 통계량을 계산하여 데이터의 분포를 파악하는 탐색적 데이터 분석 코드를 작성할 수 있다.
  • 두 개 이상의 다른 데이터 소스(예: 날씨 데이터와 판매 데이터)를 결합(Join)하여 새로운 인사이트를 도출하는 과정을 모델링할 수 있다.

태도적 목표:

  • 문제를 해결하기 위해 성급하게 코딩부터 시작하는 것이 아니라, 어떤 데이터가 필요한지 먼저 신중하게 설계하는 계획적인 태도를 기른다.
  • 데이터는 처음부터 완벽하지 않다는 것을 인지하고, 데이터의 품질을 비판적으로 검토하고 개선하려는 자세를 가진다.
  • 데이터 수집 과정에서 발생할 수 있는 저작권, 개인정보 등 법적, 윤리적 문제를 고려하는 책임감 있는 태도를 함양한다.

2. 핵심 개념 상세 설명

데이터 요구사항 정의 (Defining Data Requirements): 문제 해결의 방향을 설정하는 가장 중요한 초기 단계. 해결하려는 문제를 측정 가능하고 분석 가능한 형태로 만들기 위해, '어떤 종류의 정보가 필요한가?'를 구체적인 데이터 항목(변수, Feature)으로 정의하는 과정입니다. 예를 들어 '성적을 예측'하는 문제를 풀기 위해서는 '공부 시간', '이전 학기 성적', '수면 시간' 등의 데이터가 필요할 것입니다.

데이터 수집 (Data Collection): 정의된 요구사항에 맞는 데이터를 확보하는 과정. 데이터의 출처에 따라 다양한 방법이 사용됩니다.

  • 내부 데이터: 기업이나 조직 내부 시스템에 이미 존재하는 데이터. (예: 판매 기록, 고객 정보, 서버 로그)
  • 외부 데이터: 외부로부터 수집해야 하는 데이터.
  • API (Application Programming Interface): 공공데이터포털, 기상청, 소셜 미디어 등이 제공하는 공식적인 창구를 통해 정제된 데이터를 요청하고 받아오는 방식.
  • 웹 스크래핑/크롤링 (Web Scraping/Crawling): 웹 페이지의 HTML 구조를 분석하여 필요한 정보를 자동화된 방법으로 추출하는 기술.
  • 센서/IoT: 물리적 세계의 정보를 측정하여 디지털 데이터로 수집.
  • 설문조사/실험: 특정 목적을 위해 데이터를 직접 생성.

데이터 품질 (Data Quality) 및 정제 (Data Cleaning/Cleansing): "쓰레기를 넣으면 쓰레기가 나온다(Garbage In, Garbage Out)"는 말처럼, 데이터의 품질은 분석 결과의 품질과 직결됩니다. 수집된 원시 데이터(Raw Data)는 대부분 불완전하므로 정제 과정이 필수적입니다.

주요 품질 기준:

  • 완전성 (Completeness): 데이터에 누락된 값, 즉 **결측치(Missing Value)**가 없는가?
  • 정확성/유효성 (Accuracy/Validity): 데이터가 사실에 부합하며, 정해진 형식과 범위(예: 나이는 0 이상)에 맞는가? 비정상적으로 튀는 값인 **이상치(Outlier)**를 포함하는가?
  • 일관성 (Consistency): 동일한 사실을 나타내는 데이터가 시스템 전체에서 일관된 형태로 표현되는가? (예: '서울' vs '서울특별시')

탐색적 데이터 분석 (EDA, Exploratory Data Analysis): 수집하고 정제한 데이터를 본격적으로 모델링하거나 분석하기 전에, 데이터의 구조와 특성을 파악하기 위해 수행하는 탐색 과정. 데이터의 분포, 기초 통계량, 변수 간의 관계 등을 시각화(그래프)하거나 요약 통계를 통해 확인하며 데이터에 대한 직관과 이해를 높이는 단계입니다.

Stage 1: 무엇을 알아야 할까? (문제 정의 및 데이터 요구사항) (8분)

데이터 요구사항 정의:

  • '언제', '어디서' 사망자가 발생했는가? -> 사망자 데이터 (사망일시, 주소)

실생활 예시 코드: 이 요구사항을 Python의 자료구조로 어떻게 표현할지 설계합니다.

# 1. [데이터 요구사항 정의] 문제를 풀기 위해 필요한 데이터 구조를 설계합니다.
print("요구사항에 맞춰 필요한 데이터의 '스키마(구조)'를 정의했습니다.")

# 요구사항 1: 사망자 데이터
death_record_schema = {
    "case_id": int,      # 사례 번호
    "death_date": str,   # 사망일
    "address": str       # 사망자 주소
}
print(" - 사망자 기록:", death_record_schema)

# 요구사항 2: 주변 환경 데이터
pump_locations_schema = {
    "pump_id": str,      # 펌프 이름
    "location": (int, int) # 지도 상의 (x, y) 좌표
}
print(" - 펌프 위치 기록:", pump_locations_schema)

상세 설명: 이 코드는 프로그래밍의 시작이 코딩이 아니라 '설계'임을 보여줍니다. 실제 데이터를 수집하기 전에, 내가 어떤 형태의 데이터를 원하는지를 미리 구조(스키마)로 정의하는 과정입니다. case_id는 정수(int)로, address는 문자열(str)로 받겠다는 약속을 정함으로써, 이후의 데이터 수집과 처리 과정을 체계적으로 진행할 수 있습니다.

Stage 2: 단서를 찾아 나서다 (데이터 수집) (8분)

존 스노우가 직접 발로 뛰며 사망자 명부를 조사하고, 지도를 그려 펌프 위치를 표시한 과정을 설명합니다. (아날로그 방식의 데이터 수집)

현대라면, 공공 API나 웹 스크래핑을 통해 데이터를 수집할 수 있음을 설명합니다.

실생활 예시 코드: 관공서 웹사이트에 게시된 사망자 목록을 '웹 스크래핑'하여 수집하는 과정을 시뮬레이션합니다.

import re

def scrape_death_records_mock(html_content):
    # 2. [데이터 수집] 웹 페이지에서 필요한 정보를 추출(스크래핑)합니다.
    print("\n웹 페이지 스크래핑 시뮬레이션...")
    # 예: 
Case 101, 40 Broad Street
 형식의 텍스트에서 정보 추출
    records = []
    # 정규 표현식: Case 다음에 나오는 숫자(\d+), 그리고 쉼표 뒤의 주소(.+)를 찾음
    pattern = re.compile(r"
Case (\d+), (.+)
")
    for line in html_content.split('\n'):
        match = pattern.search(line)
        if match:
            records.append({
                "case_id": int(match.group(1)),
                "address": match.group(2)
            })
    return records

# 가상의 웹 페이지 HTML 내용
london_newspaper_html = """

Soho Outbreak Daily Report


Case 101, 40 Broad Street


Case 102, 35 Cambridge Street


Case 103, 10 Broad Street

"""
collected_data = scrape_death_records_mock(london_newspaper_html)
print(f"수집된 데이터: {collected_data}")

상세 설명: 이 코드는 데이터 수집 방법 중 '웹 스크래핑'의 원리를 보여줍니다. 실제 웹 스크래핑은 requests, BeautifulSoup 라이브러리를 사용하지만, 여기서는 핵심 원리에 집중합니다. re.search와 같은 정규 표현식을 사용하여, 정해진 패턴(Case [숫자], [주소])에 맞는 텍스트를 HTML 문서에서 찾아내어 구조화된 데이터(딕셔너리)로 추출합니다.

Stage 3: 가짜 단서 걸러내기 (데이터 품질과 정제) (7분)

수집된 주소 데이터가 "Broad St.", "Broad Street" 등으로 다르게 표기되거나, 주소가 누락된 경우가 있을 수 있음을 지적 (일관성, 완전성 문제). 분석 전에 이런 데이터를 표준화된 형태로 '정제'해야 정확한 집계가 가능함을 설명합니다.

실생활 예시 코드: 주소 표기를 통일하고, 주소가 없는 데이터를 제거하는 데이터 정제 함수를 구현합니다.

def clean_address_data(records):
    # 3. [데이터 정제] 수집된 데이터의 품질을 높입니다.
    print("\n데이터 정제 작업 시작...")
    cleaned_records = []
    for rec in records:
        # 완전성 체크: 주소 정보가 없는 데이터는 제외
        if "address" not in rec or not rec["address"]:
            print(f" -> 결측치 발견 및 제거: {rec}")
            continue
        
        # 일관성 확보: 주소 표기법 통일
        address = rec["address"].replace("St.", "Street").strip()
        rec["address"] = address
        cleaned_records.append(rec)
    return cleaned_records

messy_data = [{"address": "10 Broad St. "}, {"address": ""}, {"address": "20 Cambridge Street"}]
clean_data = clean_address_data(messy_data)
print(f"정제된 데이터: {clean_data}")

상세 설명: 이 코드는 데이터 품질을 높이는 '정제' 과정을 보여줍니다. 첫째, 주소 값이 아예 없는 결측치를 continue를 이용해 제거합니다. 둘째, .replace()와 .strip() 문자열 메서드를 사용하여 "Broad St. "와 "Broad Street"가 동일한 주소로 인식되도록 표준화합니다. 이러한 정제 과정 없이는 다음 단계의 분석에서 "Broad St."와 "Broad Street"가 서로 다른 장소로 집계되는 심각한 오류가 발생할 수 있습니다.

Stage 4: 단서 연결하기 (탐색적 데이터 분석, EDA) (7분)

존 스노우가 지도에 점을 찍어 패턴을 시각적으로 분석한 행위가 바로 EDA의 시초임을 설명. 정제된 데이터를 바탕으로, 어떤 주소에서 사망자가 가장 많이 나왔는지 '빈도수'를 계산하는 것이 가장 기본적인 분석임을 설명합니다.

수학 예시 코드: 정제된 사망자 데이터를 바탕으로, 주소별 사망자 수를 계산하고 가장 빈도가 높은 주소를 찾아냅니다.

from collections import Counter

def exploratory_data_analysis(records):
    # 4. [탐색적 데이터 분석] 데이터의 기본적인 특성을 파악합니다.
    print("\n주소별 사망자 수 분석 (EDA)...")
    # 주소만 추출하여 리스트를 만듦
    addresses = [rec["address"] for rec in records]
    # Counter로 빈도수 계산
    address_counts = Counter(addresses)
    # 가장 빈도가 높은 상위 주소 출력
    print("사망자 발생 빈도 Top 2:")
    for address, count in address_counts.most_common(2):
        print(f" - {address}: {count} 명")
    return address_counts

# 정제된 데이터라고 가정
final_data = [{'address': '40 Broad Street'}, {'address': '10 Broad Street'}, {'address': '40 Broad Street'}]
analysis_result = exploratory_data_analysis(final_data)

상세 설명: 이 코드는 '탐색적 데이터 분석(EDA)'의 핵심을 보여줍니다. Counter를 사용하여 정제된 데이터로부터 '주소별 사망자 수'라는 새로운 통계 정보를 생성했습니다. 이 분석을 통해, 우리는 40 Broad Street에서 사망자가 집중되었다는 '패턴'과 '통찰'을 얻게 됩니다. 이는 "Broad Street 주변에 콜레라의 원인이 있을 것이다"라는 가설을 세우는 결정적인 단서가 됩니다. 이처럼 EDA는 데이터의 이면에 숨겨진 이야기를 드러내는 과정입니다.

(마무리) 수업을 마치며 (5분)

문제 정의부터 데이터 요구사항, 수집, 정제, 분석으로 이어지는 체계적인 데이터 기반 문제 해결 프로세스를 정리합니다.

"중요한 것은 질문을 멈추지 않는 것이다. 호기심은 그 자체만으로도 존재 이유를 가진다." - 알베르트 아인슈타인

"여러분, 오늘 우리는 존 스노우처럼 하나의 '질문'에서 시작했습니다. 그리고 그 질문에 답하기 위해 어떤 데이터가 필요한지 고민하고, 데이터를 모으고, 정제하여 마침내 진실에 다가가는 과정을 경험했습니다. 아인슈타인의 말처럼, 위대한 발견은 정답이 아닌 '질문'에서 시작됩니다. 여러분이 앞으로 마주할 세상의 수많은 문제들에 대해, '왜 그럴까?', '어떤 데이터로 증명할 수 있을까?'와 같은 호기심 가득한 질문을 멈추지 않는 데이터 과학자로 성장하길 바랍니다. 수고 많으셨습니다."

4. 진단평가 (OX 퀴즈)

1. 데이터 기반 문제 해결에서 가장 먼저 해야 할 일은 데이터를 수집하는 것이다. (X)

상세 풀이: 가장 먼저 해야 할 일은 '풀고자 하는 문제를 명확하게 정의'하고, 그에 따라 '어떤 데이터가 필요한지(데이터 요구사항)'를 정하는 것입니다. 목표 없이 데이터를 모으는 것은 비효율적입니다.

2. 웹 스크래핑은 웹 페이지의 HTML에서 원하는 정보를 자동으로 추출하는 데이터 수집 기술이다. (O)

상세 풀이: 웹 스크래핑(또는 크롤링)은 프로그램이 웹 페이지에 접속하여, HTML 태그 구조를 분석하고 그 안의 텍스트나 링크 등의 데이터를 수집하는 기술입니다.

3. 수집된 원시 데이터(Raw Data)는 대부분 완벽하여, 별도의 정제 과정 없이 바로 분석에 사용할 수 있다. (X)

상세 풀이: 원시 데이터는 결측치, 이상치, 비일관적인 표기 등 다양한 품질 문제를 포함하는 경우가 대부분입니다. 정확한 분석을 위해서는 데이터 정제(Data Cleaning) 과정이 필수적입니다.

4. 탐색적 데이터 분석(EDA)은 데이터의 기초 통계량을 계산하거나 시각화하여 데이터의 전반적인 특징을 파악하는 과정이다. (O)

상세 풀이: EDA는 데이터에 대한 첫인상을 얻는 과정으로, 평균/중앙값/최빈값 등을 계산하거나 히스토그램/산점도 등을 그려보며 데이터의 분포와 숨겨진 패턴, 이상치 등을 탐색하는 활동입니다.

5. "학생의 최종 성적"을 예측하는 문제를 풀 때, "학생의 출석률"은 필요한 데이터 항목이 될 수 있다. (O)

상세 풀이: 학생의 성실도를 나타내는 출석률은 성적과 관련이 있을 가능성이 높습니다. 이처럼 문제 해결과 관련이 있을 것이라 예상되는 변수를 정의하는 것이 '데이터 요구사항 정의'입니다.

6. 데이터의 '완전성'은 데이터에 오타나 잘못된 값이 없는 것을 의미한다. (X)

상세 풀이: 데이터에 오타나 잘못된 값이 없는 것은 '정확성(Accuracy)'에 해당합니다. '완전성(Completeness)'은 데이터에 누락된 값, 즉 결측치가 없는 정도를 의미합니다.

7. API는 정부나 기업이 데이터를 외부에 제공하기 위해 만들어 놓은 공식적인 통로이다. (O)

상세 풀이: API는 정해진 규칙에 따라 요청을 보내면, 정제된 데이터를 약속된 형식(주로 JSON)으로 받을 수 있는 안정적이고 효율적인 데이터 수집 방법입니다.

8. "서울특별시"와 "서울"이 데이터에 섞여 있는 것은 데이터의 '일관성' 문제를 보여준다. (O)

상세 풀이: 동일한 대상을 다른 방식으로 표기하는 것은 데이터의 일관성을 해칩니다. 분석 전에 하나의 표현(예: '서울특별시')으로 통일하는 정제 작업이 필요합니다.

9. 데이터 수집 시, 다른 웹사이트의 정보를 무단으로 대량 수집하는 것은 저작권 문제가 발생할 수 있다. (O)

상세 풀이: 웹 스크래핑은 기술적으로 가능하더라도, 해당 웹사이트의 이용 약관(robots.txt 등)을 준수해야 하며, 저작권이 있는 콘텐츠를 무단으로 수집하고 상업적으로 이용하는 것은 법적 문제가 될 수 있습니다.

10. 이상치(Outlier)는 데이터의 전반적인 경향을 잘 보여주므로 분석에 반드시 포함해야 한다. (X)

상세 풀이: 이상치는 측정 오류나 특별한 이벤트로 인해 발생한, 대부분의 데이터와는 동떨어진 값입니다. 평균이나 표준편차와 같은 통계량에 큰 왜곡을 줄 수 있으므로, 분석 전에 그 원인을 파악하고 제거하거나 적절히 처리해야 하는 경우가 많습니다.

5. 형성평가 (5지선다형)

1. '배달 시간 예측' 모델을 만들기 위해 데이터를 수집하려고 한다. 다음 중 데이터 요구사항으로 가장 거리가 먼 것은?

  1. 주문 시간대
  2. 배달 거리
  3. 날씨 (비, 눈 등)
  4. 배달 기사의 이름
  5. 음식 종류 (조리 시간)

정답: ④

상세 풀이: 배달 시간은 주문 시간(교통량), 거리, 날씨, 조리 시간 등과 깊은 관련이 있습니다. 하지만 특정 배달 기사의 '이름' 자체는 배달 시간을 예측하는 데 직접적인 변수가 되기 어렵습니다. (물론 '기사의 숙련도'는 변수가 될 수 있지만, 이름은 식별자일 뿐입니다.)

2. 공공데이터포털에서 제공하는 '전국 미세먼지 측정 정보'를 내 프로그램으로 가져오고 싶을 때, 가장 적합하고 일반적인 데이터 수집 방법은?

  1. 웹 스크래핑
  2. API 호출
  3. 설문조사
  4. 직접 측정
  5. 서버 로그 분석

정답: ②

상세 풀이: 공공데이터포털과 같이 데이터를 공식적으로 제공하는 기관은 대부분 외부 개발자들이 쉽고 안정적으로 데이터를 활용할 수 있도록 API를 제공합니다. 웹사이트의 화면 구조를 분석해야 하는 스크래핑보다 훨씬 효율적이고 안정적입니다.

3. 수집된 나이 데이터 [21, 23, 22, 19, -5, 25] 에서 발견되는 가장 명백한 데이터 품질 문제는?

  1. 불완전성 (결측치)
  2. 비일관성
  3. 부정확성 (유효하지 않은 값)
  4. 중복성
  5. 낮은 활용성

정답: ③

상세 풀이: 나이는 음수 값을 가질 수 없습니다. -5는 명백히 잘못 입력되었거나 오류로 생성된 '유효하지 않은 값'으로, 데이터의 '정확성'에 문제가 있음을 보여줍니다.

4. 데이터 기반 문제 해결의 첫 번째 단계인 '문제 정의'에서 가장 중요한 활동은?

  1. 어떤 프로그래밍 언어를 사용할지 결정하기
  2. 해결하고자 하는 목표와 범위를 명확히 하기
  3. 데이터를 저장할 데이터베이스 종류를 선택하기
  4. 데이터 시각화에 사용할 그래프 종류를 정하기
  5. 알고리즘의 시간 복잡도를 계산하기

정답: ②

상세 풀이: '문제 정의' 단계에서는 기술적인 세부사항보다, '우리가 풀어야 할 문제가 정확히 무엇인가?', '성공의 기준은 무엇인가?'와 같이 프로젝트의 근본적인 목표와 방향을 설정하는 것이 가장 중요합니다.

5. 정제된 데이터를 본격적으로 분석하기 전에, 데이터의 분포, 이상치 존재 여부, 변수 간의 관계 등을 시각적으로 확인하며 데이터에 대한 이해도를 높이는 과정을 무엇이라고 하는가?

  1. 데이터 수집 (Data Collection)
  2. 데이터 모델링 (Data Modeling)
  3. 탐색적 데이터 분석 (Exploratory Data Analysis)
  4. 알고리즘 설계 (Algorithm Design)
  5. 시스템 배포 (System Deployment)

정답: ③

상세 풀이: EDA는 데이터와 '첫 만남'을 갖는 과정입니다. 복잡한 분석에 앞서 데이터가 어떻게 생겼는지, 어떤 특징을 가졌는지 대략적으로 훑어보며 분석 방향에 대한 직관과 아이디어를 얻는 중요한 탐색 단계입니다.

6. "키가 175cm 이상이고, 몸무게가 70kg 미만인 사람 찾기" 라는 문제를 풀기 위해, '키' 데이터와 '몸무게' 데이터를 모두 수집했다. 이 과정은 다음 중 무엇에 해당하는가?

  1. 데이터 정제
  2. 데이터 요구사항 정의
  3. 알고리즘 설계
  4. 탐색적 데이터 분석
  5. 데이터 시각화

정답: ②

상세 풀이: '키'와 '몸무게'는 문제를 풀기 위해 반드시 필요한 데이터 항목(변수)입니다. 이처럼 어떤 데이터가 필요한지를 결정하고 정의하는 과정이 바로 '데이터 요구사항 정의'입니다.

7. 수집된 센서 데이터에 누락된 값(결측치)이 많을 때, 이를 처리하는 방법으로 적절하지 않은 것은?

  1. 결측치가 포함된 행 전체를 삭제한다.
  2. 다른 데이터들의 평균값이나 중앙값으로 대체한다.
  3. 결측치를 0으로 일괄 대체한다.
  4. 결측치를 그대로 둔 채 분석을 진행한다.
  5. 머신러닝 모델을 이용해 가장 가능성 높은 값으로 예측하여 채운다.

정답: ④

상세 풀이: 대부분의 분석 알고리즘과 라이브러리는 결측치를 처리하지 못하고 오류를 발생시킵니다. 따라서 결측치를 그대로 두고 분석을 진행하는 것은 거의 불가능하며, 분석 결과의 왜곡을 유발합니다. ①, ②, ③, ⑤는 모두 상황에 따라 사용되는 결측치 처리 방법입니다.

8. 다음 중 데이터 수집 시 법적/윤리적 문제를 야기할 가능성이 가장 높은 행위는?

  1. 기상청 API를 통해 공개된 날씨 데이터를 수집하는 행위
  2. 웹사이트의 'robots.txt' 정책을 무시하고 과도하게 웹 스크래핑하는 행위
  3. 연구 목적으로 진행한 설문조사 응답 데이터를 분석하는 행위
  4. 내가 직접 촬영한 우리 집 고양이 사진 데이터를 수집하는 행위
  5. 공공데이터포털에 공개된 지하철 이용객 통계 데이터를 수집하는 행위

정답: ②

상세 풀이: 'robots.txt'는 웹사이트 운영자가 크롤러(수집 봇)에게 수집을 허용하거나 금지하는 페이지를 명시한 국제적인 규약입니다. 이를 무시하고 과도하게 데이터를 수집하는 행위는 웹사이트 서버에 부하를 주어 업무 방해에 해당할 수 있으며, 저작권 침해 등의 법적 문제로 이어질 수 있습니다.

9. 존 스노우가 런던 지도 위에 사망자 발생 위치를 점으로 표시하여 패턴을 분석한 행위는 현대의 데이터 과학에서 어떤 활동과 가장 유사한가?

  1. 데이터 암호화
  2. 데이터 압축
  3. 데이터 시각화 (Data Visualization)
  4. 데이터베이스 설계
  5. 펌웨어 개발

정답: ③

상세 풀이: 지도 위에 데이터를 표시하여 특정 지역(브로드 가 펌프)에 데이터가 집중되는 현상을 한눈에 파악한 것은, 데이터를 그래프나 맵 등으로 표현하여 패턴과 인사이트를 찾는 '데이터 시각화'의 고전적인 사례라고 할 수 있습니다.

10. 문제 해결을 위한 데이터 분석 프로젝트를 수행하는 올바른 마음가짐은?

  1. 데이터는 항상 정확하므로, 수집된 데이터를 의심 없이 사용한다.
  2. 내가 세운 초기 가설과 데이터 분석 결과가 다르면, 결과를 가설에 맞춰 수정한다.
  3. 데이터 수집부터 분석, 결과 해석까지 전 과정에서 발생 가능한 오류와 편향을 비판적으로 검토한다.
  4. 복잡한 최신 알고리즘을 사용하는 것만이 항상 최고의 결과를 보장한다.
  5. 분석 결과는 나만 알고 있는 것이 가장 가치 있다.

정답: ③

상세 풀이: 데이터 과학자는 '과학자'로서의 태도를 가져야 합니다. 즉, 데이터와 분석 과정을 끊임없이 의심하고, 객관적인 증거를 찾으며, 자신의 편견이 결과에 영향을 미치지 않도록 경계하는 비판적 사고가 매우 중요합니다.

6. 코딩 실습 예제

1. 데이터 요구사항 정의하기 (서술형)

문제: '우리 반 학생들의 수면 시간이 성적에 미치는 영향'을 분석하는 프로젝트를 수행한다고 가정하자. 이 문제를 해결하기 위해 수집해야 할 데이터 항목(변수)들을 3가지 이상 구체적으로 정의하고, 각 데이터의 타입을 명시하세요.

예시 답안:

  • 평균 수면 시간 (avg_sleep_hours): 분석의 핵심 독립 변수. 하루 평균 몇 시간을 자는지 기록. (타입: 실수(Float), 예: 7.5)

  • (추가) 하루 평균 공부 시간 (avg_study_hours): 성적에 영향을 미칠 수 있는 중요한 제3의 변수. (타입: 실수(Float))
  • (추가) 주말/주중 구분 (is_weekend): 주말과 주중의 수면 패턴이 다를 수 있으므로 구분. (타입: 불리언(Boolean) 또는 'weekday'/'weekend' 문자열)

상세 풀이 및 설명: 이 활동은 코딩 없이, 논리적으로 문제를 해결하는 첫 단계를 연습합니다. '성적에 미치는 영향'이라는 막연한 문제를 '수면 시간'과 '시험 성적'이라는 측정 가능한 두 변수의 관계로 구체화했습니다. 또한, '공부 시간'과 같이 결과에 함께 영향을 줄 수 있는 다른 변수(혼란 변수)를 고려함으로써 더 정교한 분석을 위한 데이터 요구사항을 정의할 수 있음을 보여줍니다.

2. API 데이터 수집 시뮬레이션

특정 도시의 날씨 정보를 제공하는 가짜 API 함수 get_weather_api(city_name)를 만드세요. 이 함수는 도시 이름을 받아, {"city": city_name, "temp": 25.5, "humidity": 60} 와 같은 딕셔너리를 반환하도록 작성하세요.

import random
import json

def get_weather_api(city_name):
    """특정 도시의 날씨 정보를 제공하는 가상 API 함수."""
    print(f"API 호출: '{city_name}'의 날씨 정보를 요청합니다...")
    # 실제 API는 네트워크 통신을 하지만, 여기서는 데이터 생성을 시뮬레이션
    if city_name in ["서울", "부산", "제주"]:
        weather_data = {
            "city": city_name,
            "temp_c": round(random.uniform(10.0, 30.0), 1),
            "humidity_percent": random.randint(40, 80),
            "condition": random.choice(["맑음", "구름 많음", "비"])
        }
        return json.dumps(weather_data, ensure_ascii=False)
    else:
        return json.dumps({"error": "City not found"})

# 테스트
seoul_weather_json = get_weather_api("서울")
print(f"API 응답 (JSON): {seoul_weather_json}")

# JSON 파싱하여 사용
seoul_weather_data = json.loads(seoul_weather_json)
print(f"파싱 결과: 서울의 현재 온도는 {seoul_weather_data.get('temp_c')}°C 입니다.")

상세 풀이 및 설명: 이 코드는 외부 서비스(날씨 서버)로부터 데이터를 가져오는 과정을 시뮬레이션합니다. get_weather_api 함수가 API의 '엔드포인트' 역할을 합니다. 실제 상황에서는 이 함수를 호출하는 것이 http://api.weather.com/seoul 과 같은 URL에 접속하는 것에 해당합니다. API는 데이터를 체계적으로 전달하기 위해 JSON 형식의 문자열을 반환하는 경우가 일반적입니다. json.dumps가 이 역할을 합니다. API를 사용하는 측에서는 이 JSON 문자열을 json.loads로 파싱하여 딕셔너리로 변환한 뒤, 필요한 값(temp_c)을 키로 접근하여 사용합니다.

3. 데이터 타입 변환 (정제)

['10.5', '20.1', 'Error', '30.3'] 와 같이 숫자와 오류 문자열이 섞인 리스트가 있습니다. 이 리스트를 순회하며, 숫자로 변환 가능한 값은 float으로 변환하고, 변환이 불가능한 값은 리스트에서 제거하는 clean_and_convert_to_float(data) 함수를 작성하세요.

def clean_and_convert_to_float(data_list):
    """문자열 리스트에서 숫자로 변환 가능한 값만 float으로 변환합니다."""
    cleaned_data = []
    print(f"원본 데이터: {data_list}")
    for item in data_list:
        try:
            # 문자열을 float으로 변환 시도
            float_value = float(item)
            cleaned_data.append(float_value)
        except (ValueError, TypeError):
            # 변환 중 오류 발생 시 (숫자가 아니거나 None일 때), 해당 항목 무시
            print(f" -> '{item}'은(는) 유효한 숫자가 아니므로 제외합니다.")
    return cleaned_data

# 테스트
sensor_readings_str = ['10.5', '20.1', 'Invalid Reading', '30.3', None, '15.8']
numeric_readings = clean_and_convert_to_float(sensor_readings_str)
print(f"정제 및 변환 후 데이터: {numeric_readings}")

상세 풀이 및 설명: 이 코드는 데이터 정제의 핵심 과정 중 하나인 '타입 변환'과 '오류 데이터 처리'를 보여줍니다. 웹 스크래핑 등으로 수집된 데이터는 숫자로 보여도 실제로는 문자열 타입인 경우가 많습니다. try-except 구문은 오류 처리에 매우 유용합니다. try 블록 안의 float(item) 코드를 실행하다가, 만약 item이 숫자로 변환될 수 없는 문자열("Invalid Reading")이거나 None이면 ValueError 또는 TypeError가 발생합니다. except 블록은 이 에러를 '잡아서' 프로그램이 중단되는 것을 막고, 오류가 발생했음을 알린 뒤 다음 데이터로 넘어가도록 합니다. 이를 통해 오염된 데이터 속에서 유효한 데이터만 안전하게 추출할 수 있습니다.

4. 결측치 개수 세기

[1, 2, None, 4, None, 6] 와 같이 결측치(None)가 포함된 리스트를 받아, 전체 데이터의 개수, 결측치의 개수, 그리고 결측치의 비율을 출력하는 report_missing_values(data) 함수를 작성하세요.

def report_missing_values(data):
    """데이터 리스트의 결측치(None) 통계를 보고합니다."""
    print(f"\n--- 데이터 품질 보고서 (총 {len(data)}개) ---")
    total_count = len(data)
    if total_count == 0:
        print("데이터가 없습니다.")
        return
    
    # 리스트의 .count() 메서드를 사용하여 None의 개수를 직접 셈
    missing_count = data.count(None)
    
    if missing_count == 0:
        print("결측치가 발견되지 않았습니다. (완전성: 100%)")
    else:
        missing_ratio = (missing_count / total_count) * 100
        print(f"전체 데이터 개수: {total_count}")
        print(f"결측치 개수: {missing_count}")
        print(f"결측치 비율: {missing_ratio:.2f}%")

# 테스트
dataset1 = [1, 2, 3, 4, 5]
dataset2 = [10, 20, None, 40, None, 60, 70, None]
report_missing_values(dataset1)
report_missing_values(dataset2)

상세 풀이 및 설명: 데이터의 '완전성(Completeness)'을 파악하는 것은 분석의 신뢰도를 결정하는 중요한 첫걸음입니다. 이 함수는 데이터셋에 얼마나 많은 정보가 누락되었는지를 정량적으로 보여줍니다. data.count(None): 파이썬 리스트의 .count() 메서드는 특정 값이 리스트 안에 몇 번 나타나는지 세어주는 편리한 기능입니다. 결측치 비율이 너무 높을 경우(예: 50% 이상), 해당 데이터셋은 분석에 사용하기 어렵다고 판단하거나, 데이터 수집 과정을 다시 점검해야 한다는 의사결정을 내릴 수 있습니다.

5. 기초 통계량 계산 (EDA)

숫자 데이터 리스트를 받아, 해당 데이터의 개수, 최솟값, 최댓값, 평균값을 계산하여 딕셔너리 형태로 반환하는 calculate_basic_stats(data) 함수를 작성하세요.

def calculate_basic_stats(data):
    """숫자 리스트의 기초 통계량(개수, 평균, 최소, 최대)을 계산합니다."""
    if not data:
        return {"error": "데이터가 비어있습니다."}
    
    stats = {
        "count": len(data),
        "mean": sum(data) / len(data),
        "min": min(data),
        "max": max(data)
    }
    return stats

# 테스트
scores = [85, 92, 78, 65, 100, 88, 95, 72, 90, 81]
score_stats = calculate_basic_stats(scores)

print("--- 시험 성적 탐색적 데이터 분석(EDA) 결과 ---")
print(f" - 학생 수: {score_stats['count']} 명")
print(f" - 평균 점수: {score_stats['mean']:.2f} 점")
print(f" - 최고 점수: {score_stats['max']} 점")
print(f" - 최저 점수: {score_stats['min']} 점")

상세 풀이 및 설명: 이 코드는 '탐색적 데이터 분석(EDA)'의 가장 기본적인 활동을 보여줍니다. 파이썬의 내장 함수 len(), sum(), min(), max()를 사용하여 복잡한 라이브러리 없이도 데이터의 핵심적인 특징을 빠르게 파악할 수 있습니다. 이러한 기초 통계량은 데이터의 전반적인 분포를 이해하는 데 도움을 줍니다. 예를 들어, 평균은 84.6점인데 최저점이 65점이라면, 점수 분포가 비교적 고르게 퍼져있음을 짐작할 수 있습니다. 만약 최저점이 10점이라면, 이상치가 존재할 가능성을 의심해볼 수 있습니다.

6. 데이터 조인(Join) 시뮬레이션

학생 정보 딕셔너리 `students = {"1": "Alice", "2": "Bob"}` 와 성적 정보 딕셔너리 `scores = {"1": 95, "2": 88}`가 있습니다. 이 두 딕셔너리를 학생 ID를 기준으로 합쳐, `[{'id': '1', 'name': 'Alice', 'score': 95}, ...]` 형태의 리스트를 만드는 프로그램을 작성하세요.

def merge_student_data(student_dict, score_dict):
    """학생 ID를 기준으로 학생 정보와 성적 정보를 병합(조인)합니다."""
    merged_list = []
    # 학생 정보 딕셔너리를 기준으로 반복
    for student_id, name in student_dict.items():
        # 성적 딕셔너리에서 해당 학생의 점수를 찾음
        score = score_dict.get(student_id)
        # 두 데이터 소스에 모두 정보가 있는 경우에만 병합
        if score is not None:
            merged_list.append({
                "id": student_id,
                "name": name,
                "score": score
            })
    return merged_list

# 테스트
# 서로 다른 곳에서 수집된 데이터라고 가정
students = {"S001": "Alice", "S002": "Bob", "S003": "Charlie"}
scores = {"S001": 95, "S003": 87, "S004": 91} # Bob의 성적은 누락, S004는 학생 명부에 없음

full_data = merge_student_data(students, scores)
print("--- 분리된 데이터 조인 결과 ---")
print(full_data)

상세 풀이 및 설명: 실제 문제 해결에서는 여러 다른 출처의 데이터를 결합해야 하는 경우가 많습니다. 이를 '조인(Join)' 또는 '병합(Merge)'이라고 합니다. 이 코드는 관계형 데이터베이스의 JOIN 연산을 시뮬레이션합니다. '학생 ID'가 두 데이터를 연결하는 공통된 '키(key)' 역할을 합니다. for 루프로 하나의 데이터 소스(학생 정보)를 순회하면서, 다른 데이터 소스(성적 정보)에서 해당 키를 가진 정보를 찾아 결합합니다. `score_dict.get(student_id)`를 사용하여, 성적 정보가 없는 학생(Bob)의 경우에도 오류 없이 처리하도록 했습니다. 최종 결과에는 두 데이터 소스에 모두 존재하는 학생(Alice, Charlie)의 정보만 포함됩니다.

7. 이상치 제거 (범위 기반)

온도 데이터 리스트에서 정상 범위를 벗어나는(예: 0도 미만 또는 40도 초과) 값들을 이상치로 간주하고, 이를 제거한 새로운 리스트를 반환하는 `remove_outliers_by_range(data, min_val, max_val)` 함수를 작성하세요.

def remove_outliers_by_range(data, min_val, max_val):
    """정상 범위를 벗어나는 이상치를 제거합니다."""
    print(f"정상 범위: {min_val} ~ {max_val}")
    # 리스트 컴프리헨션을 사용해 정상 범위에 속하는 값만 필터링
    cleaned_data = [value for value in data if min_val <= value <= max_val]
    # 제거된 이상치 찾기 (설명을 위해)
    outliers = [value for value in data if value < min_val or value > max_val]
    if outliers:
        print(f"제거된 이상치: {outliers}")
    return cleaned_data

# 테스트
temperatures = [22.5, 23.1, -50.0, 22.8, 24.0, 100.0, 23.5]
# 정상적인 대기 온도의 물리적 범위를 앎 (도메인 지식)
valid_range_min = -40.0
valid_range_max = 50.0
cleaned_temps = remove_outliers_by_range(temperatures, valid_range_min, valid_range_max)
print(f"원본: {temperatures}")
print(f"이상치 제거 후: {cleaned_temps}")

상세 풀이 및 설명: 이 코드는 '도메인 지식(Domain Knowledge)'을 활용한 데이터 정제 방법을 보여줍니다. 통계적인 방법 외에도, 우리는 해당 데이터가 의미하는 것(여기서는 '대기 온도')에 대한 사전 지식을 가지고 있습니다. 대기 온도가 -50℃나 100℃가 되는 것은 일반적인 상황이 아니므로, 이는 명백한 오류(이상치)라고 판단할 수 있습니다. `remove_outliers_by_range` 함수는 이렇게 사전에 정의된 '정상 값의 범위'를 기준으로 데이터를 필터링합니다. `[value for value in data if min_val <= value <= max_val]`: 이 리스트 컴프리헨션은 data 리스트의 모든 value에 대해 min_val과 max_val 사이에 있는지 검사하고, 조건을 만족하는 value만 모아 새로운 리스트를 만드는 매우 효율적이고 간결한 파이썬 구문입니다.

8. 데이터 일관성 확보 (표준화)

['Seoul', 'Busan', 'seoul', 'Incheon', 'SEOUL'] 과 같이 대소문자가 섞여있는 도시 이름 리스트가 있습니다. 모든 도시 이름을 소문자로 통일하고, 고유한 도시 이름 목록을 출력하는 프로그램을 작성하세요.

def standardize_and_get_unique(items):
    """데이터를 표준화(소문자, 공백제거)하고, 고유한 값만 추출합니다."""
    # 1. 표준화: 모든 항목을 소문자로 바꾸고 앞뒤 공백을 제거
    standardized_items = [item.lower().strip() for item in items]
    print(f"표준화 후: {standardized_items}")
    # 2. 집합(set)으로 변환하여 중복을 자동으로 제거
    unique_items = set(standardized_items)
    # 3. 다시 리스트로 변환하여 반환 (순서는 보장되지 않음)
    return list(unique_items)

# 테스트
cities = ['Seoul', 'Busan', 'seoul', 'Incheon', ' SEOUL', 'busan']
print(f"원본 데이터: {cities}")
unique_cities = standardize_and_get_unique(cities)
print(f"일관성 처리 후 고유한 도시 목록: {unique_cities}")

상세 풀이 및 설명: 이 코드는 데이터의 일관성(Consistency) 문제를 해결하는 방법을 보여줍니다. `item.lower().strip()`: `lower()`는 문자열을 모두 소문자로, `strip()`은 앞뒤의 불필요한 공백을 제거합니다. 이 두 작업을 통해 'Seoul', 'seoul', ' SEOUL ' 등은 모두 동일한 'seoul'로 표준화됩니다. `set(standardized_items)`: 파이썬의 `set`은 중복된 원소를 허용하지 않는 자료구조입니다. 리스트를 `set`으로 변환하는 순간, 모든 중복된 항목들이 사라지고 고유한 값들만 남게 됩니다. 이 두 가지 기법을 조합하면, 일관성 없는 데이터를 쉽고 효율적으로 정제할 수 있습니다.

9. 파생 변수 생성

[{'id': 1, 'birth_year': 2005}, {'id': 2, 'birth_year': 2006}] 와 같은 학생 데이터 리스트가 있습니다. 현재 연도를 기준으로 각 학생의 '나이(age)'를 계산하여 기존 딕셔너리에 'age' 키와 함께 추가하는 `add_age_feature(student_list, current_year)` 함수를 작성하세요.

from datetime import date

def add_age_feature(student_list, current_year):
    """학생 데이터에 'birth_year'를 이용해 'age'라는 파생 변수를 추가합니다."""
    print("--- 'age' 파생 변수 생성 ---")
    for student in student_list:
        if "birth_year" in student:
            age = current_year - student["birth_year"] # 한국 나이 X, 만 나이
            student["age"] = age # 기존 딕셔너리에 새로운 키-값 쌍 추가
    return student_list

# 테스트
students = [
    {'id': 'S001', 'name': '뉴턴', 'birth_year': 1643},
    {'id': 'S002', 'name': '아인슈타인', 'birth_year': 1879}
]
# 현재 연도 가져오기
this_year = date.today().year
students_with_age = add_age_feature(students, this_year)
print(students_with_age)

상세 풀이 및 설명: 파생 변수(Derived Feature) 또는 특성 공학(Feature Engineering)은 기존에 있던 데이터들을 조합하거나 가공하여, 문제 해결에 더 도움이 되는 새로운 변수를 만드는 과정입니다. '출생 연도' 자체보다 '나이'가 성적이나 다른 행동을 분석하는 데 더 직관적이고 유용한 변수가 될 수 있습니다. `add_age_feature` 함수는 각 학생의 `birth_year`를 가져와 간단한 계산(`current_year - birth_year`)을 통해 `age`라는 새로운 정보를 만들어냅니다. `student["age"] = age`: 이 코드는 원본 딕셔너리에 `age`라는 새로운 키와 계산된 값을 직접 추가합니다. 이처럼 기존 데이터를 가공하여 새로운 특성을 만들어내는 것은 머신러닝 등 데이터 분석에서 매우 중요한 단계입니다.

10. 간단한 데이터 분석 프로젝트: 영화 평점 분석

[{'title': '영화A', 'rating': 9.5}, ...] 와 같은 영화 평점 데이터가 있습니다. 평점이 9.0 이상인 '추천 영화' 목록을 만들고, 전체 영화의 평균 평점을 계산하여 출력하는 프로그램을 작성하세요.

def analyze_movie_ratings(movies):
    """영화 평점 데이터를 분석하여 추천 목록과 평균 평점을 구합니다."""
    # 1. 문제 정의: 평점이 9.0 이상인 영화를 찾고, 전체 평균 평점을 계산
    # 2. 데이터 요구사항: 영화 제목, 평점
    if not movies:
        return "분석할 데이터가 없습니다."
        
    # 3. 알고리즘 & 4. 구현
    recommended_titles = []
    total_rating = 0.0
    
    for movie in movies:
        rating = movie.get('rating', 0.0) # 결측치 대비
        total_rating += rating
        # 추천 영화 선정 로직
        if rating >= 9.0:
            recommended_titles.append(movie.get('title', '제목 없음'))
            
    # 평균 계산
    average_rating = total_rating / len(movies)
    
    # 5. 결과 정리 및 출력
    print("--- 영화 평점 분석 결과 ---")
    print(f"전체 영화 수: {len(movies)}편")
    print(f"평균 평점: {average_rating:.2f} 점")
    print(f"평점 9.0 이상 추천 영화: {recommended_titles}")

# 테스트 데이터
movie_data = [
    {'title': '인터스텔라', 'rating': 9.4},
    {'title': '기생충', 'rating': 9.2},
    {'title': '어벤져스: 엔드게임', 'rating': 8.8},
    {'title': '매트릭스', 'rating': 9.0},
    {'title': '조커', 'rating': 8.5}
]
analyze_movie_ratings(movie_data)

상세 풀이 및 설명: 이 코드는 '데이터 분석 프로젝트'의 축소판으로, 데이터로부터 정보를 추출하여 사용자의 의사결정(어떤 영화를 볼지)에 도움을 주는 과정을 보여줍니다. 데이터 수집: `movie_data` 리스트가 이미 수집된 데이터 역할을 합니다. 데이터 처리 및 분석: `for` 루프를 통해 각 영화 데이터를 순회합니다. `total_rating += ...`: 전체 평균을 구하기 위해 평점을 누적(집계)합니다. `if rating >= 9.0`: 특정 조건(규칙)에 따라 '추천 영화'를 필터링합니다. 결과 제시: 분석을 통해 얻어진 정보('평균 평점', '추천 영화 목록')를 사용자에게 보기 쉽게 요약하여 제공합니다. 이는 데이터 분석의 최종 목표인 '가치 있는 정보 제공'에 해당합니다.
