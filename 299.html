개념적 목표:

  • 데이터 분석 프로젝트의 전체 수명 주기(문제 정의 → 데이터 수집 → 데이터 정제 → 탐색적 분석 → 모델링/분석 → 결과 공유)를 설명할 수 있다.
  • 프로젝트의 목표에 따라 필요한 데이터를 정의하고, 적절한 분석 및 시각화 기법을 선택하는 과정의 중요성을 이해한다.
  • 데이터 정제, 탐색적 데이터 분석(EDA), 특성 공학(Feature Engineering)이 최종 분석 결과의 품질에 미치는 영향을 설명할 수 있다.
  • 분석 결과를 해석하여 의미 있는 '인사이트'와 '스토리'를 도출하고, 이를 통해 설득력 있는 주장을 펼치는 과정을 이해한다.

절차적 목표:

  • Pandas를 사용하여 실제 데이터셋(CSV 파일 등)을 불러오고, .info(), .describe(), .isnull().sum() 등을 이용해 데이터의 기본 구조와 품질을 파악할 수 있다.
  • 결측치 처리, 데이터 타입 변환, 파생 변수 생성 등 기본적인 데이터 정제 및 가공 작업을 수행할 수 있다.
  • .groupby()를 이용한 그룹별 집계와 Matplotlib를 이용한 시각화(막대, 선, 산점도 등)를 통해 탐색적 데이터 분석을 수행하고, 데이터에서 패턴을 발견할 수 있다.
  • 분석의 전 과정을 논리적인 순서에 따라 코드로 구현하고, 각 단계의 결과와 해석을 주석이나 마크다운으로 기록하여 하나의 완성된 분석 보고서(또는 노트북)를 작성할 수 있다.

태도적 목표:

  • 막연한 문제를 구체적이고 검증 가능한 질문으로 정의하고, 이를 해결하기 위해 체계적으로 계획을 수립하는 과학적 탐구 자세를 가진다.
  • 분석 과정에서 나타나는 예상치 못한 결과나 데이터의 한계를 비판적으로 수용하고, 객관적인 사실에 기반하여 결론을 도출하려는 자세를 기른다.
  • 자신의 분석 결과를 명확한 근거(데이터와 시각화)를 바탕으로 다른 사람에게 효과적으로 전달하고 공유하려는 책임감 있는 소통 능력을 함양한다.

2. 핵심 개념 상세 설명

데이터 분석 프로젝트 수명 주기 (Data Analysis Project Lifecycle): 하나의 데이터 분석 프로젝트가 아이디어에서부터 최종 보고서까지 완성되는 표준적인 과정. 일반적으로 다음과 같은 단계를 포함하며, 각 단계는 서로 영향을 주고받으며 반복적으로 수행될 수 있습니다.

  • 문제 정의 및 계획 (Problem Definition & Planning): 가장 중요한 첫 단계. 비즈니스 또는 연구 목표를 이해하고, 이를 데이터로 답할 수 있는 구체적인 질문으로 정의합니다. (예: "어떤 시간대에 자전거 대여량이 가장 많은가?") 성공의 기준과 프로젝트 계획을 수립합니다.
  • 데이터 수집 (Data Collection): 정의된 문제를 해결하는 데 필요한 데이터를 확보합니다. 내부 데이터베이스, 공공 API, 웹 스크래핑 등 다양한 방법을 사용합니다.
  • 데이터 준비 및 정제 (Data Preparation & Cleaning): 수집된 원시 데이터(Raw Data)를 분석에 적합한 형태로 가공하는 과정. 결측치 처리, 이상치 탐지, 데이터 타입 변환, 데이터 표준화 등이 포함되며, 전체 프로젝트 시간의 60~80%를 차지할 만큼 중요하고 시간이 많이 소요되는 작업입니다.
  • 탐색적 데이터 분석 (EDA, Exploratory Data Analysis): 데이터를 시각화하거나 기초 통계량을 계산하며 데이터에 숨겨진 패턴, 관계, 구조 등을 탐색하는 과정. 이 과정을 통해 데이터에 대한 깊은 이해를 얻고, 본격적인 분석 방향에 대한 가설을 수립합니다.
  • 모델링 및 심층 분석 (Modeling & In-depth Analysis): EDA를 통해 얻은 가설을 검증하거나, 미래를 예측하기 위해 통계적 모델 또는 머신러닝 모델을 구축하고 분석을 수행하는 단계.
  • 결과 공유 및 발표 (Result Sharing & Presentation): 분석을 통해 발견한 결론과 인사이트를 차트, 그래프, 대시보드, 보고서 등의 형태로 정리하여, 의사결정권자나 대중이 쉽게 이해할 수 있도록 전달하는 최종 단계. 데이터 스토리텔링 역량이 중요합니다.
  • 특성 공학 (Feature Engineering): 기존에 주어진 변수(특성, Feature)들을 조합하거나 가공하여, 모델의 성능을 높이거나 분석에 더 유용한 새로운 변수를 만들어내는 과정. (예시: '대여일시'라는 데이터가 있을 때, '요일', '시간대', '주중/주말'과 같은 새로운 파생 변수를 만들어내면 요일별, 시간대별 분석이 가능해집니다.)

Stage 3: 데이터 다듬기 (데이터 준비 및 정제) (8분)

'대여시간' 열을 문자열에서 datetime 객체로 변환하여 시간 관련 분석을 용이하게 합니다. datetime 객체에서 '시간(hour)'이라는 파생 변수를 생성합니다. (특성 공학) 결측치나 이상치가 있는지 확인하고 처리하는 과정을 설명합니다.

실생활 예시 코드: 대여시간에서 'hour' 열을 추출하고, 가상의 결측치를 처리합니다.

# 3. [데이터 정제 및 가공]
print("\n데이터 정제 및 '시간' 파생 변수 생성...")
# '대여시간' 열을 datetime 타입으로 변환 (이미 위에서 변환됨)
df['대여시간'] = pd.to_datetime(df['대여시간'])

# '시간' 파생 변수 생성
df['시간'] = df['대여시간'].dt.hour

print("결측치 확인:")
print(df.isnull().sum())
# (만약 결측치가 있다면) df.dropna() 또는 df.fillna(0) 등으로 처리

print("\n가공 후 데이터 (시간 열 추가):")
print(df.head())

상세 설명: 원시 데이터는 종종 분석에 바로 쓰기 어려운 형태입니다. '2023-10-26 08:05'라는 문자열보다, 시간 계산이 가능한 datetime 객체가 훨씬 유용합니다. `df['대여시간'].dt.hour`는 datetime 객체에서 시간 부분만 쉽게 추출하는 Pandas의 강력한 기능입니다. 이렇게 '시간'이라는 새로운 특성을 만들어내는 것이 바로 **특성 공학(Feature Engineering)**의 예시입니다.

Stage 4: 데이터 속 이야기 찾기 (탐색적 데이터 분석, EDA) (10분)

앞서 정의한 분석 질문에 답하기 위한 시각화를 수행합니다. 시간대별 이용량은 선 그래프, 인기 대여소는 막대그래프로 시각화합니다.

실생활 예시 코드: 시간대별 대여량을 계산하고 선 그래프로 시각화합니다.

# 4. [탐색적 데이터 분석 (EDA)]
print("\n분석 질문 1: 시간대별 이용량은 어떻게 되는가?")

# 시간대별 대여 건수 계산 (groupby)
hourly_counts = df.groupby('시간').size()

# 시각화
plt.figure(figsize=(10, 6))
hourly_counts.plot(kind='line', marker='o', grid=True)
plt.title('시간대별 따릉이 대여량', fontsize=16)
plt.xlabel('시간 (시)', fontsize=12)
plt.ylabel('대여 건수', fontsize=12)
plt.xticks(range(0, 24))
plt.show()

print("그래프 해석: 출퇴근 시간대인 8시와 18시에 대여량이 집중되는 패턴(인사이트)을 발견할 수 있습니다.")

상세 설명: EDA는 가설을 시각적으로 확인하는 과정입니다. "출퇴근 시간에 사람이 많을 것이다"라는 가설을 확인하기 위해, `.groupby('시간').size()`를 이용해 시간대별 건수를 집계합니다. 이를 선 그래프로 그리자, 8시와 18시 부근에 뚜렷한 피크(peak)가 나타나는 것을 볼 수 있습니다. 이 시각화 자료는 가설을 강력하게 뒷받침하는 '증거'가 되며, '출퇴근 시간대에 자전거를 추가 배치해야 한다'는 의사결정으로 이어질 수 있는 중요한 '인사이트'입니다.

print("\n[전체 요약 정보]")
df.info()

상세 풀이 및 설명: df.head(n): 데이터의 첫 n개 행을 보여줍니다. 데이터가 어떻게 생겼는지 파악하는 데 가장 먼저 사용됩니다. df.tail(n): 데이터의 마지막 n개 행을 보여줍니다. 데이터가 순서대로 정렬되어 있을 경우 유용합니다. df.info(): 데이터프레임의 전체적인 정보를 요약해줍니다. 각 열의 이름, 결측치가 아닌 데이터의 개수(Non-Null Count), 데이터 타입(Dtype), 메모리 사용량 등을 보여주어 데이터의 구조와 품질을 한눈에 파악하게 해줍니다. 'item'과 'quantity' 열에 결측치가 있음을 알 수 있습니다.

2. 기술 통계량 확인

3. 결측치 처리

문제: item 열의 결측치는 "Unknown"으로, quantity 열의 결측치는 해당 열의 평균값으로 채워 넣고, 처리 후 결측치가 없는지 다시 확인하세요.

print("\n--- 3. 결측치 처리 ---")
print("처리 전 결측치 개수:\n", df.isnull().sum())
# 'item' 열 결측치 처리
df['item'].fillna('Unknown', inplace=True)
# 'quantity' 열 결측치 처리
mean_qty = df['quantity'].mean()
df['quantity'].fillna(mean_qty, inplace=True)
print("\n처리 후 결측치 개수:\n", df.isnull().sum())
print("\n처리 후 데이터프레임:")
print(df)

상세 풀이 및 설명: .fillna(value, inplace=True): 결측치(NaN, None)를 지정된 value로 채우는 메서드입니다. `inplace=True` 옵션은 원본 데이터프레임을 직접 수정하라는 의미입니다. 문자열 열은 "Unknown", "N/A" 등 특정 문자열로 채우고, 숫자형 열은 평균값(mean)이나 중앙값(median)으로 채우는 것이 일반적인 결측치 처리 방법입니다.

4. 파생 변수 생성 (특성 공학)

문제: 'date' 열에서 '요일' 정보를 추출하여 'day_of_week'라는 새로운 열을 추가하세요.

print("\n--- 4. 파생 변수 생성 ---")
df['day_of_week'] = df['date'].dt.day_name()

상세 풀이 및 설명: Pandas에서 datetime 타입의 열은 `.dt` 접근자를 통해 다양한 시간 관련 정보(연, 월, 일, 요일, 시간 등)에 쉽게 접근할 수 있습니다. `df['date'].dt.day_name()`: 'date' 열의 각 날짜에 해당하는 요일 이름을 문자열(e.g., 'Monday')로 반환합니다. 이렇게 생성된 'day_of_week' 열은 "어떤 요일에 판매량이 높은가?"와 같은 새로운 분석을 가능하게 하는 중요한 파생 변수가 됩니다.

5. 조건에 따른 데이터 필터링

print("\n--- 5. 조건에 따른 데이터 필터링 ---")
is_fruit = df['category'] == '과일'
is_many = df['quantity'] >= 10
filtered_df = df[is_fruit & is_many]

상세 풀이 및 설명: Pandas의 불리언 인덱싱(Boolean Indexing)을 사용한 필터링입니다. `df['category'] == '과일'`은 각 행의 'category'가 '과일'이면 True, 아니면 False인 시리즈를 만듭니다. `&` 연산자를 사용하여 두 개의 불리언 시리즈에 대해 AND 연산을 수행합니다. `df[...]` 안에 최종 불리언 시리즈를 넣으면, True인 행들만 선택되어 새로운 데이터프레임이 생성됩니다.

10. 분석 결과 보고 (데이터 스토리텔링)

문제: 지금까지 분석한 내용을 바탕으로, "과일과 채소 중 어떤 카테고리의 평균 가격이 더 높은가?"라는 질문에 대한 답을 시각화 자료와 함께 텍스트로 설명하세요.

print("\n--- 10. 분석 결과 보고 ---")
# 분석 (6번 예제 재사용)
avg_price_by_cat = df.groupby('category')['price'].mean()
# 시각화
avg_price_by_cat.plot(kind='barh', figsize=(8, 4), grid=True)
plt.title('카테고리별 평균 가격 비교')
plt.xlabel('평균 가격 (원)')
plt.ylabel('카테고리')
plt.show()
# 텍스트 설명 (스토리텔링)
fruit_price = avg_price_by_cat['과일']
veg_price = avg_price_by_cat['채소']
print("\n[분석 결론]")
print(f"분석 결과, '채소' 카테고리의 평균 가격({veg_price:.0f}원)이 '과일' 카테고리의 평균 가격({fruit_price:.0f}원)보다 더 높은 것으로 나타났습니다.")
print("시각화된 막대그래프를 통해 두 카테고리의 가격 차이를 직관적으로 확인할 수 있습니다.")

상세 풀이 및 설명: 데이터 분석 프로젝트의 최종 단계는 '결과 전달'입니다. 이 코드는 단순히 그래프만 그리는 것이 아니라, 그 그래프가 무엇을 의미하는지 명확한 텍스트로 함께 설명합니다. 데이터 스토리텔링: ① 질문 제기("어떤 카테고리가 더 비싼가?") → ② 분석 및 근거 제시 (groupby로 평균 계산, barh(수평 막대그래프)로 시각화) → ③ 결론 도출 ("채소가 더 높다"). 이처럼 분석 결과와 시각화를 결합하여 하나의 논리적인 이야기를 만드는 것이 데이터 분석 프로젝트의 성공적인 마무리입니다.
