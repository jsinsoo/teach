과학고 정보 수업: 목표지향적 개념기반 수업지도안

수업 주제: III. 알고리즘과 프로그래밍 - 322. 모델링: 현실을 코드로 번역하는 추상화의 기술

수업 시작 멘트

지도를 만드는 사람은 세상의 모든 나무와 돌멩이를 그리지 않습니다. 오직 길, 강, 도시와 같은 핵심적인 요소만을 추려내어, 우리가 세상을 이해하고 길을 찾을 수 있도록 돕습니다. 프로그래밍에서의 '모델링'은 바로 이 지도 제작과 같습니다. 복잡한 현실 세계에서 문제 해결에 필요한 본질만을 꿰뚫어 보고, 그것을 컴퓨터가 이해할 수 있는 단순하고 우아한 구조로 표현하는 기술이죠. 오늘 우리는 현실을 코드로 번역하는 위대한 지도 제작자, 즉 모델러가 되어 보겠습니다.

1. 학습 목표

개념적 목표

  • 모델링이 현실 세계의 문제나 개념을, 컴퓨터가 처리할 수 있도록 핵심적인 속성과 행동으로 '단순화'하고 '구조화'하는 과정임을 설명할 수 있다.
  • 추상화(Abstraction)가 모델링 과정에서 불필요한 세부사항을 제거하고 문제의 본질에 집중하는 핵심 원리임을 이해한다.
  • 객체지향의 클래스가 데이터(속성)와 기능(메서드)을 하나로 묶어 현실 세계를 모델링하는 효과적인 도구임을 설명할 수 있다.

절차적 목표

  • 주어진 문제 상황을 분석하여, 모델링에 필요한 핵심적인 속성과 메서드를 식별할 수 있다.
  • 식별된 내용을 바탕으로 파이썬 class를 사용하여 데이터 모델을 설계하고 구현할 수 있다.
  • 생성된 객체들을 리스트나 딕셔너리와 같은 데이터 구조에 담아 관리하는 코드를 작성할 수 있다.

태도적 목표

  • 복잡한 현상을 체계적으로 분석하고 핵심을 파악하려는 과학적 사고방식을 기른다.
  • 코드를 작성하기 전에, '어떻게 표현할 것인가'를 먼저 고민하는 설계 중심의 접근 방식을 내재화한다.
  • 잘 만들어진 모델(클래스)이 코드의 가독성, 재사용성, 확장성을 어떻게 향상시키는지 체감한다.

2. 핵심 개념 상세 설명

모델링 (Modeling)

현실 세계의 사물, 시스템, 개념 등을 컴퓨터 프로그램 내에서 표현하기 위해, 그 본질적인 특징을 뽑아내어 단순화하고 구조화하는 과정입니다. 잘 된 모델링은 복잡한 문제를 이해하고 해결하는 것을 더 쉽게 만듭니다. 예를 들어, 기상 예측 시스템은 복잡한 대기 현상을 온도, 습도, 기압 등의 변수와 물리 법칙(방정식)으로 모델링한 것입니다.

추상화 (Abstraction)

모델링의 핵심적인 사고 과정입니다. 어떤 대상을 표현할 때, 문제 해결에 필요하지 않은 불필요하고 세부적인 정보는 과감히 '숨기거나 무시'하고, 현재 문제와 관련된 '핵심적인 속성과 기능'에만 집중하는 것입니다. 우리는 자동차를 운전할 때, '가속 페달'과 '핸들'이라는 추상화된 인터페이스를 사용할 뿐, 엔진 실린더의 움직임이나 전자 제어 장치의 신호까지 알 필요가 없습니다.

객체지향 모델링 (Object-Oriented Modeling)

현실 세계의 대상을 '객체(Object)'로 모델링하는 방식입니다. 여기서 객체란, 관련된 데이터(속성)와 그 데이터를 처리하는 기능(메서드)을 하나로 묶은 소프트웨어 단위를 의미합니다.

  • 클래스(Class) as a Model: 객체지향 프로그래밍에서 클래스는 특정 종류의 객체를 만들기 위한 '설계도'이자, 현실의 개념을 담아내는 '모델' 그 자체입니다.
  • 속성(Attribute): 모델링하려는 대상이 가지는 '데이터' 또는 '상태'. (예: '학생' 모델의 '이름', '학년', '점수')
  • 메서드(Method): 모델링하려는 대상이 수행할 수 있는 '행동' 또는 '기능'. (예: '학생' 모델의 '공부하다', '시험보다', '평균 점수 계산하다')

3. 수업의 흐름 상세 기술

1단계: 도입 (5분) - 원자 모델의 진화

"톰슨의 건포도 푸딩 모델, 러더퍼드의 행성 모델, 보어의 궤도 모델... 과학자들은 눈에 보이지 않는 '원자'의 실체를 이해하기 위해, 관측된 사실을 가장 잘 설명하는 다양한 '모델'을 만들어왔습니다. 각 모델은 불필요한 복잡성을 제거하고 핵심적인 특징(원자핵, 전자)에 집중하는 '추상화'의 결과물이었죠. 프로그래밍도 마찬가지입니다. 우리는 '클래스'라는 도구를 사용하여, 우리가 해결하려는 문제의 핵심 개념을 가장 잘 표현하는 '모델'을 직접 설계하고 만들어야 합니다."

2단계: 전개 (35분) - 화학 원소 주기율표 모델링 프로젝트

활동 1: 문제 이해 및 분석 - 무엇을 모델링할 것인가?

문제: "화학 원소들의 정보를 관리하는 프로그램을 만들고 싶다. 각 원소는 원자 번호, 이름, 원소 기호, 원자량을 가진다. 이 정보를 바탕으로, 특정 원소의 정보를 쉽게 찾아보고, 새로운 원소를 추가할 수 있어야 한다."

  • 핵심 개념(명사) 식별: '화학 원소(Element)' → 클래스의 좋은 후보.
  • 속성(데이터) 식별: '원자 번호', '이름', '원소 기호', '원자량' → Element 클래스의 속성이 된다.
  • 행동(기능) 식별: '정보를 보여주다', '새로운 원소를 추가하다'(이것은 관리 프로그램의 기능) → Element 클래스의 메서드 또는 관리 프로그램의 함수가 된다.

활동 2: 설계 - Element 클래스 구조화

분석한 내용을 바탕으로 Element 클래스를 의사코드와 함께 설계합니다.

"우리가 만들 Element 객체는 어떤 정보를 가지고 태어나야 할까요? 생성자(__init__)에서 원자 번호, 이름, 기호, 원자량을 받아 자신의 속성으로 저장해야 합니다. 그리고 이 객체를 print했을 때, 정보가 보기 좋게 출력되면 좋겠습니다. __str__ 메서드가 필요하겠네요."

클래스 구현


# 1. 클래스 설계: Element라는 개념을 모델링
class Element:
    # 2. 생성자 구현: 객체가 생성될 때 필요한 속성을 정의
    def __init__(self, atomic_number, name, symbol, atomic_mass):
        # self.속성 = 값 형태로 객체의 상태를 저장
        self.atomic_number = atomic_number
        self.name = name
        self.symbol = symbol
        self.atomic_mass = atomic_mass

    # 3. 행동(메서드) 구현: 객체의 정보를 보기 좋게 표현하는 기능
    def __str__(self):
        # f-string을 사용하여 속성들을 조합한 문자열을 반환
        return (f"[{self.symbol}] {self.name} (No. {self.atomic_number})\n"
                f" - Mass: {self.atomic_mass}")

# 4. 모델(클래스)을 사용한 객체(인스턴스) 생성
hydrogen = Element(1, "Hydrogen", "H", 1.008)
helium = Element(2, "Helium", "He", 4.0026)

# 5. 객체의 메서드(기능) 사용
print(hydrogen)
print(helium)
이 Element 클래스는 바로 '화학 원소'라는 현실 세계 개념에 대한 우리의 '모델'입니다. 우리는 이 설계도만 있으면 어떤 원소 객체든 찍어낼 수 있습니다.

활동 3: 모델의 활용 - 주기율표(데이터 구조) 만들기

"이제 이 원소 객체들을 어떻게 관리할까요? 여러 개의 객체를 담는 데는 리스트나 딕셔너리가 적합합니다."

# 딕셔너리를 사용하여 원소 기호로 빠르게 찾을 수 있는 주기율표 모델링
periodic_table = {}

# 데이터 추가
element_data = [
    (1, "Hydrogen", "H", 1.008),
    (2, "Helium", "He", 4.0026),
    (6, "Carbon", "C", 12.011),
    (8, "Oxygen", "O", 15.999),
]

for num, name, sym, mass in element_data:
    # Element 클래스로 객체를 만들어, 딕셔너리에 추가
    periodic_table[sym] = Element(num, name, sym, mass)

# 모델 활용: 'C'(탄소)의 정보 찾아보기
carbon_element = periodic_table['C']
print("\n--- 탄소 정보 조회 ---")
print(carbon_element)
보세요. periodic_table이라는 딕셔너리에 우리가 직접 설계한 Element 객체들을 담았습니다. 이제 우리는 periodic_table['C'] 처럼 매우 직관적인 방법으로 탄소 객체를 찾고, 그 객체의 정보(print)를 얻을 수 있습니다. 이것이 바로 잘 된 모델링의 힘입니다.

3단계: 정리 (5분) - 추상화와 현실

모델링이 현실을 100% 똑같이 복제하는 것이 아니라, 문제 해결에 필요한 '핵심'만을 추출하는 '추상화' 과정임을 다시 한번 강조합니다. "우리의 Element 클래스는 양성자, 중성자, 전자 궤도와 같은 복잡한 내용은 모두 생략했습니다. 왜냐하면 '원소 정보 관리'라는 현재 문제에서는 불필요하기 때문입니다. 좋은 모델은 항상 문제의 '맥락'에 따라 달라집니다."

수업 마무리 멘트

천재 건축가 안토니 가우디는 자연의 구조에서 영감을 얻어 사그라다 파밀리아 성당을 설계했습니다. 그는 나무가 어떻게 하중을 지탱하는지, 벌집의 구조가 얼마나 효율적인지를 '모델링'하여 자신의 건축물에 적용했습니다. 여러분은 오늘 현실 세계의 개념을 '클래스'라는 구조로 모델링하는 법을 배웠습니다. 앞으로 여러분이 마주할 모든 문제는 여러분만의 모델링을 기다리는 미지의 건축물과 같습니다. 그 본질을 꿰뚫어 보고 가장 효율적이고 아름다운 구조를 설계해내는, 코드의 가우디가 되시기를 바랍니다.

4. 진단 및 형성평가

진단평가 (OX 퀴즈 10개)

1. 모델링은 현실 세계의 대상을 컴퓨터 코드로 똑같이 복제하는 과정이다. (X)

풀이: 모델링은 똑같이 복제하는 것이 아니라, 문제 해결에 필요한 핵심적인 특징만을 뽑아내어 '단순화'하고 '구조화'하는 과정입니다.

2. '추상화'는 문제 해결에 불필요한 세부 사항은 숨기는 것을 의미한다. (O)

풀이: 이것이 추상화의 핵심 정의입니다. 복잡성을 줄이고, 문제의 본질에 집중하게 해줍니다.

3. 객체지향 프로그래밍에서, class는 모델링을 위한 가장 중요한 도구이다. (O)

풀이: 클래스는 데이터(속성)와 기능(메서드)을 하나로 묶어, 현실 세계의 개념을 하나의 독립적인 객체로 모델링하는 데 사용되는 핵심적인 문법입니다.

4. 문제 설명에 나오는 '명사'는 주로 클래스의 '메서드'가 된다. (X)

풀이: 일반적으로 명사(Nouns)는 객체를 나타내는 '클래스'나 객체의 데이터를 나타내는 '속성(attribute)'의 후보가 됩니다. '동사(Verbs)'가 객체의 행동인 '메서드'가 됩니다.

5. 하향식 설계는 전체 시스템에서 시작하여 더 작은 기능 단위로 나누어 가는 방식이다. (O)

풀이: 하향식(Top-down) 설계는 가장 크고 추상적인 목표에서 시작하여, 이를 달성하기 위해 필요한 하위 기능들로 점차 분해해 나가는 설계 방식입니다.

6. "학생은 이름을 가진다"에서 '이름'은 Student 클래스의 메서드가 된다. (X)

풀이: '이름'은 학생의 상태를 나타내는 데이터이므로, '속성(attribute)', 즉 self.name이 되어야 합니다.

7. 잘 된 모델링은 코드의 재사용성을 높여준다. (O)

풀이: Element 클래스를 한 번 잘 만들어두면, 주기율표 프로그램, 분자량 계산 프로그램 등 원소 정보가 필요한 어떤 프로젝트에서든 이 클래스를 재사용할 수 있습니다.

8. 문제를 여러 개의 작은 함수나 클래스로 나누면, 각 부분을 개별적으로 테스트하기 더 쉬워진다. (O)

풀이: 이는 모듈화의 큰 장점 중 하나입니다. 각 부품(함수, 클래스)이 의도대로 정확히 동작하는지 독립적으로 테스트(단위 테스트, Unit Test)할 수 있어, 전체 시스템의 안정성을 높일 수 있습니다.

9. C++의 struct는 여러 데이터를 하나로 묶는다는 점에서, 파이썬 클래스의 데이터 모델링 기능과 유사하다. (O)

풀이: C++의 struct는 관련 데이터들을 하나의 구조로 묶는 역할을 합니다. 비록 메서드를 포함하는 완전한 클래스와는 차이가 있을 수 있지만, 데이터를 구조화하여 모델링한다는 근본적인 목적은 파이썬 클래스와 매우 유사합니다.

10. 모델링은 코딩을 시작하기 전에 수행하는, 문제 해결을 위한 계획 단계이다. (O)

풀이: 성공적인 프로그래밍은 코딩에 앞서 어떤 데이터와 기능이 필요하고, 이들을 어떻게 구조화할지 설계하는 데서 시작됩니다. 모델링은 이 설계 과정의 핵심입니다.

형성평가 (5지선다형 10개)

1. "온라인 쇼핑몰" 시스템을 객체지향적으로 설계할 때, 클래스로 모델링하기 가장 부적절한 것은?

  1. Customer (고객)
  2. Product (상품)
  3. Order (주문)

문제 5: 위 Student 클래스에, 모든 수강 과목의 평균 점수를 계산하는 get_average_score() 메서드를 추가하세요.

class Student:
    # ... __init__, add_subject ...
    def __init__(self, name):
        self.name = name
        self.subjects = []
    def add_subject(self, name, score):
        self.subjects.append(Subject(name, score))

    def get_average_score(self):
        if not self.subjects:
            return 0.0
        
        total_score = sum([subj.score for subj in self.subjects])
        return total_score / len(self.subjects)

s1 = Student("뉴턴")
s1.add_subject("고전역학", 100)
s1.add_subject("광학", 95)
print(f"{s1.name}의 평균 점수: {s1.get_average_score()}") # 97.5

문제 7: 중력이 있는 '자유 낙하 물체'를 모델링하세요. FallingObject 클래스는 mass, position`, velocity를 속성으로 가집니다. 1초의 시간이 흐르는 것을 시뮬레이션하는 step() 메서드를 구현하세요. (단, 중력가속도 g=9.8, 가속도 a = g)

class FallingObject:
    def __init__(self, mass, initial_position=1000.0):
        self.mass = mass
        self.position = initial_position
        self.velocity = 0.0 # 초기 속도는 0
        self.g = 9.8

    def step(self, time_delta=1.0):
        # v = v0 + at
        self.velocity += self.g * time_delta
        # s = s0 - v*t (아래로 떨어지므로)
        self.position -= self.velocity * time_delta
        if self.position < 0:
            self.position = 0

    def __str__(self):
        return f"위치: {self.position:.2f}m, 속도: {self.velocity:.2f}m/s"

ball = FallingObject(mass=10)
print(f"t=0s, {ball}")
ball.step()
print(f"t=1s, {ball}")
ball.step()
print(f"t=2s, {ball}")
상세한 풀이와 설명: 물리 현상을 객체지향적으로 모델링하는 예제입니다. 물체의 상태(위치, 속도)를 속성으로 정의하고, 시간이 지남에 따라 상태가 변하는 물리 법칙을 step() 메서드로 구현합니다.

문제 8: 간단한 'DNA 서열' 클래스를 만드세요. __init__에서 서열 문자열을 받고, 'GC 함량'(전체 서열 중 G와 C의 비율)을 계산하여 반환하는 gc_content() 메서드를 구현하세요.

class DNASequence:
    def __init__(self, sequence):
        # 입력된 서열을 대문자로 통일하여 저장
        self.sequence = sequence.upper()

    def gc_content(self):
        if not self.sequence:
            return 0.0
        g_count = self.sequence.count('G')
        c_count = self.sequence.count('C')
        return (g_count + c_count) / len(self.sequence) * 100

class Vehicle:
    def __init__(self):
        self.speed = 0
    
    def accelerate(self):
        self.speed += 5
        print(f"가속! 현재 속도: {self.speed}")

class Car(Vehicle):
    def __init__(self, brand):
        super().__init__() # 부모의 생성자 호출하여 speed 속성 초기화
        self.brand = brand
    
    # 메서드 오버라이딩
    def accelerate(self):
        self.speed += 10 # 부모보다 더 빠르게 가속
        print(f"{self.brand} 가속! 현재 속도: {self.speed}")

generic_vehicle.accelerate()

my_car.accelerate()
